I plan to add support to the tcl server to support multiline tcl
commands and replies.

The question then is what char the multiline command
should end with.

Taking my inspiration from Linux, I'm thinking ctrl-d might
be a good choice for command sent and reply returned...

This will obviously be incompatible, but I believe the
tcl server as-is would have seen relatively little use,
so I'm more concerned about making the right choice
here than backwards compatibility.

Thoughts?
