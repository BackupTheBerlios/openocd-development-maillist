Thanks to everyone that provided feedback about the status of OpenOCD's
support for Cortex-A8. I have made some notes and started hacking on
some low-hanging fruit. Proper support for ROM Table scanning to
discover the debug topology seems to be high on the list of necessary
improvements to push Cortex-A8 support toward being "complete", but a
review of the code has led me to other possible improvements.

However, my primary strategic goal will be to add initial support for
the Cortex-A9 core and SMP chips using it (e.g. OMAP4, U8500). I am
trying to put together a full game plan this week, but I would again
appreciate any feedback from those who have gone before me. Has anyone
else been planning to attack this new problem?
