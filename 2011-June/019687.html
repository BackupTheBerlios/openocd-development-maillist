UrJtag can use libusb-1.0 API or libusb-0.1 API. Right now OpenOCD
uses libusb-0.1 API.

UrJtag can use either libftdi-1.0 or libftdi-0.1x. When using libftdi-1.0,
it can make use of the async interface and speed up operation. I
am not so sure if it uses the async capability of libusb-1.0.

So for OpenOCD, it may take times to move to really use libusb-1.0
API and libftdi-1.0's async feature. And my request is not in this
aspect, but rather to be able to configure with using libftdi-1.0.

To do that, then it needs to detect libftdi-1.0 ( and as a result, need
to detect libusb-1.0 as well).
