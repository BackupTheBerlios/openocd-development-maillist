This implies that configure.in is missing.  bootstrap needs to be run  
in the main source directory even if you plan to build out of tree.   
bootstrap just builds configure which is typically part of the release  
process and not done by end users.  Once you've run bootstrap,  
configure can be run out of tree.

> sh: ../configure: No such file or directory
