The berlios site has a bug database configured for OpenOCD. There are even
some bugs registered. AFAIK, that bugdatabase is OK if not fantastically
cool like Bugzilla.

I guess the *real* problem is that there are nobody working on those bugs :-)

I find that 70-90% of the work required to resolve a bug *can* be done by
the best testers, but when testers simply leave a note that things aren't
working(5-10% of the work involved in resolving the issue) then that
is unlikely to
cause the problem to be addressed and the bug will, even if real, just clutter
the bug database.

Basically I think that for all bugs where the tester has exhausted the
suggestions
below(done the things that make sense and could possibly be relevant/useful)
should be tracked somehow. The rest can live in the mailing list in the hope
that someone takes an interest.

http://svn.berlios.de/svnroot/repos/openocd/trunk/BUGS
