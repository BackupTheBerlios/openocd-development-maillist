Hello list,

PRESTO keeps TCK high between its commands (which is a necessity),
however, that means PRESTO may even pull reset line while TCK is high.
As this might cause problems, according to note about TCK idle state in bitbang.c,
the attached patch fixes it - TCK returns to zero before state of reset line is changed.

Please commit, regards,
   Pavel
