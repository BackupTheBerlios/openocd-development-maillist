Hi List,

"run_and_init" clears the CPU registers to their default state, e.g. PC = 0x00000000, but "run_and_halt" does not. Is this the intended behaviour?

I assumed that both options could be used to reset a CPU to it's default state if it could not be halted using hardware watchpoints at address 0x00000000 and using SRST (because SRST also pull TRST) as observed on AT91SAM7 series.
