Hi all!

A while ago I looked into the possibility of adding some form of SWD support
for Cortex-M3 to OpenOCD. I used an Amontec JTAGkey Tiny with an external
tristate buffer (74'126) controlled by nTRST to multiplex TDI and TDO onto
the bi-directional SWDIO pin. After the better part of a weekend it worked
to the point of allowing me to do basic downloading and debugging of an
STM32. There seem to be some issues, mainly related to reset/startup and
error recovery.

The difficult part was to force the SWD protocol onto the existing
jtag-centric API. It probably would have been better suited as a separate
module but I was short on time so I hijacked the ft2232 driver and the
existing jtag command queue. Relatively few changes to cortex_swjdp.c were
needed to handle the protocol/register differences. The attached patch shows
the result. I haven't tried if it still works in jtag mode. The code itself
is more or less an ugly hack but could serve as a basis for discussion on
how to add a proper SWD infrastructure. Sorry for all the removed whitespace
at the end of lines, courtesy of Eclipse.

Regards,
Andreas Fritiofson
