A thought in this direction is to use Doxygen to generate  
documentation from source code comments.  It keeps the documentation  
close to the code so it is more likely to be up to date and the final  
output form can be similar to what we have today.  Some areas of the  
docs, like the getting started section, could still be in separate  
files, but it means that things like a list of targets could be  
generated from the source.
