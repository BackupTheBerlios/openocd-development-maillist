Yes, the main intent of Doxygen is to build documentation from the  
source code.  You _can_ abuse it, however, to build documentation that  
isn't about the API.  It may not be worth the trouble, but is an  
option.  Instead, we could use a TCL script that asks for the  
available target types and options and generates the texi format for  
that section of the manual.
