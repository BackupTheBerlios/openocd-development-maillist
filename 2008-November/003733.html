Speaking from the experience of coordinating releases for large and  
small software projects, there is never a version that everyone is  
happy with.  You need to pick a point where all the new features that  
you want in are in, make a branch, and then fix the branch until it is  
stable enough for release.  Since this is a 1.0, any set of features  
is sufficient.  If we make a 1.0 branch, we can work on fixing bugs in  
that branch while new features (the new jtag tap syntax for example)  
are developed in trunk.
