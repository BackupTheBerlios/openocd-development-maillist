Spen napsal(a):
> we are not removing wait_halt as such (just renaming it), this will become
> the default behaviour for halt.
 > we could always keep the name wait_halt (mark as depreciated) and it will
 > just call the new halt.

Just a note for clarification:
wait_halt does NOT halt the target. It just wait for the target to halt itself,
but the reason for the halt has to come from elsewhere. So renaming it
or marking it deprecated does not really make a sense. This command is really useful for scripting.
All what is needed to be done is to call wait_halt as a subroutine when halt command is processed,
so probably move part of the code handle_wait_halt_command to separate function.
and call it as subroutine from both halt and wait_halt.
