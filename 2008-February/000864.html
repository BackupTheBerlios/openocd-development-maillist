I've got a target where the working area moves after the MMU has been
initialized, but working_area takes a single fixed address which means
I can't specify a working area that works before and after the MMU has
been set up.

Did I miss something?

This is awkward as I would like working memory before and after the MMU
has been initialized.

After a bit of pondering, I can see how this is non-trivial to "fix" by
e.g. changing working_area to take the physical address. The best
I have for now is to make working_area COMMAND_ANY instead
of COMMAND_CONFIG.

Translating from virtual to physical is unambigious and available via
the "xscale virt2phys" command. The reverse translation is trickier
and ambigious(pick the first?).

Thoughts?
