On Fri, Feb 15, 2008 at 5:01 PM, Jonathan Larmour <jifl at ecoscentric.com> wrote:
> Øyvind Harboe wrote:
>  > I've got a target where the working area moves after the MMU has been
>  > initialized, but working_area takes a single fixed address which means
>  > I can't specify a working area that works before and after the MMU has
>  > been set up.
>  >
>  > Did I miss something?
>  >
>  > This is awkward as I would like working memory before and after the MMU
>  > has been initialized.
>
>  One other possibility: temporarily disable the MMU before doing an
>  operation in the working area, and restore it to its previous state after.

A couple of problems with that:

- the parameters to the code to be executed from the working area will
be in virtual address space. Worse the memory to which the algorithm
refers might not even be contiguous.
- caches will be disabled. Minimally this will make the code slower.
