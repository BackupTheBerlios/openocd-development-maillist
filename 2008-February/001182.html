Committed.

Embedded operating systems often ship with oodles of drivers.

This patch, proof of concept, show shows how those drivers can
be used with OpenOCD in the case that OpenOCD does not
support a particular flash type.

In the example attached an eCos flash driver supports
AT91EB40a.

NB!!!! Even if the driver comes from eCos, the developer
is not in any way obliged to use or learn about eCos.

It's not done, but it's time to get some input:

- drivers should be part of the library that OpenOCD ships with,
just like the target config script library
- it would be *very* handy if the drivers could support relocation(via
position independent code?) and arbitary flash base addresses
