> IMHO we'd loose some flexibility for little gain, other than saving one line per halt in a script.

halt 0 would be identical to todays behaviour, so no flexibility lost.

The gain is less confusion. The problem can be illustrated by GDB
scripts like this:

monitor halt
monitor xxx
monitor yyy

will be flaky. Most of the time the halt will be done by the time the
second monitor
command arrives.

The conclusion is of course that OpenOCD is "flaky" because the
developer is confused
about what's going on.
