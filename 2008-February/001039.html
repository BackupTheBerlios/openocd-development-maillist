Hello Michael,

I don't think it's a good idea to use printf for anything. Currently it's very 
easy to reroute all output from the OpenOCD by simply modifying the log 
macros.

I think we all agree that file, line and time information isn't important at 
all for the version and url, but we should probably introduce a new log type, 
rather than forcing output to stdout like we would when using printf.

Any ideas or comments?
