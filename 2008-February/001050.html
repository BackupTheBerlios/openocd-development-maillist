It's *great* to have these things documented. If we disagree
about something w.r.t. the jtag_xxx() API, then in any case
we should *first* document the current expected behaviour
and rules, then perhaps switch rules and update the
codebase to respect some new and improved rule....

The alternative to adding your rules above, would be to require
the users of the jtag_xxx() API to use jtag_add_pathmove()
in those cases where the above is the required behaviour.
However, that becomes *tricky* to enforce.

Perhaps jtag_add_statemove() should actually *invoke*
jtag_add_pathmove() for the above cases? For now
I think documenting the requirements and requiring any
implementation of jtag_xxx() to respect the API is good enough.
