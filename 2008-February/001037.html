On Saturday 23 February 2008 17:47:03 Pavel Chromy wrote:
> Hello list,
>
> there is a bug in server.c, remove_connection()
> which does not update linked list pointers correctly
> upon list member removal.
> Actually, it always updates pointer to the head of the list
> instead the next field of the previous member.
> The patch corrects this using double pointer technique.
>
> Similar problem affect also remove_service() which does not update
> pointers at all. In fact this function never gets called, but corrected in
> the patch anyway.
>
> The patch also modifies counterpart functions add_service and
> add_connection to use the same double pointer technique to make it
> consistent.
> Moreover the code is more compact this way.
> Commit?
>
> Note: the bug only reveals when more than one connection to a single
> service is made, typically GDB with mutliple targets, and a later
> established connection gets closed first.
>
> Regards,
>    Pavel

Hello Pavel,

comitted, thanks for the patch.
