Hi List, Øyvind and Michael, 

I agree that the script naming convention should be standardised. I guess that there will be a lot of different personal preferences, but standardising will be a good thing.

Coming from a M$ Windoze working environment, I'm partial to using the extension *.ocd for OpenOCD scripts. *.gdb is already used for GDB scripts. Maybe "*_dev.ocd", "*_cfg.ocd", "*_scr.ocd" ... can be used for the naming convention?

Attached are files that I would like to share with the list. In the "at91sam7s_armusbtiny.ocd" you will see that I tried to follow the OpenOCD documentation order. This makes it easier to look up what each setting is for.

That's my 2 cents worth.

Best regards and thanks for all the great work!

Pieter
