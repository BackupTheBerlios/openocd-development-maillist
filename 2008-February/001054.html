On Sun, Feb 24, 2008 at 7:20 PM, Dominic Rath <Dominic.Rath at gmx.de> wrote:
> Hello list,
>
>  I'm planning to add the following to the OpenOCD README (some of it
>  shamelessly stolen from U-Boot's README):
>
>  ----[snip]----------------------------------------------------------------------
>  5. Coding Style
>
>  The following rules try to describe formatting and naming conventions that
>  should be followed to make the whole OpenOCD code look more consistent.
>  The ultimate goal of coding style should be readability, and these rules may
>  be ignored for a particular (small) piece of code if that makes it more
>  readable.
>
>  Formatting rules:
>  - remove any trailing white space
>  - use TAB characters for indentation, not spaces
>  - displayed TAB width is 4 characters

This would require text editors to be configured as 8 chars
is what text editors would have as default?

It also wouldn't show correctly in browser when displaying e.g.
patches, I think.

Personally I've got a white-space disorder. It's kinda like dyslexia.
My brain disregards all white-space. This can cause a problem
when submitting patches, but not when reading code.  I do my
best, but white space is as difficult for me as spelling is
for someone with dyslexia.

Other than that I don't think the formatting rules should
cause much problems for anyone.
