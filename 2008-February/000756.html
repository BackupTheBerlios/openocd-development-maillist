> Hi Øyvind,
>
> Does "wait_halt" wait until the whole reset script has been executed, or just until
> the target has been halted and in a reset state?

I honestly don't know what it does, nor does the documentation define it,
which is precisely why I want the reset command to define the behaviour :-)

Here is my working list on what changes I'd like to see to reset:

- make it synchronous
- reset takes two arguments, action + timeout. Default action is "reset run",
  the target->reset_mode is removed.
- the target command defines a reset_mode argument. This argument will
  be deprecated.
- if target->reset_mode cannot be removed, then minimally "reset" should
  not modify target->reset_mode.

Comments?
