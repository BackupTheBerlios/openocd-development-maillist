Regarding the resethandling patch.
This breaks flash programming when using a reset script to program a target.
Only happens with reset mode reset_init not run_and_init, as
flash_init_drivers have already been called.

This was noticed on targets that can do a reset @ 0x0, eg. arm9, cortex.

Do we really need the reset patch?

configs/jtag lines.
