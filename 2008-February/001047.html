I think the way to go here is to switch to a log_add/remove_callback()
scheme where there can be any number of listeners and the code that
invokes log_add_callback() is responsible to invoke the
log_remove_callback(). OpenOCD naming convention can be respected as
part of that rewrite.

How does that sound? Would you like to have a stab at it? :-)

>  4) In general, I am not against pushing log messages to the user, but the current implementation is weird.
>  It is probably fine to have this for telnet connection, but I am not convinced that it is really a good idea
>  for GDB.

Those developers that use GDB monitor commands in lieu of a telnet
session should have the same output as in a telnet session. Currently
GDB monitor will miss out on the asynchronous messages, but I think
that's acceptable.

Admittedly the implementation needs a log_add/remove_callback()
fix, but is an implementation issue.
