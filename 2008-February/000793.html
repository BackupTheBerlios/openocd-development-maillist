Concerning scripting and possible errors or timeouts:
There is currently no way to process part of the script conditionally, AFAIK.
This is typically not a problem, but it would be pretty useful to have possibility for handling.
"exceptions". I mean something like "on_timeout break/continue" and "on_error break/continue" commands
to interrupt script processing if something bad had happened
(like in some implementations of BASIC in those ancient times)
I am not sure whether is it possible to implement this without significant effort but this would
definitely be a nice feature.
