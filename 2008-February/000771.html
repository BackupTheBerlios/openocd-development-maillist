I like the idea, but it does break any flash programming called from a reset
script - if reset_init is used
As you know the flash drivers have not been registered yet, an poll will
call any callbacks when the target halts.

reset_init is used a lot on ARM9, cortex targets as they can handle reset @
0x0 correctly.
