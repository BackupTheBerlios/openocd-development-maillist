> Openocd already will supply gdb the memory map if enabled, and you have a
> xml enabled gdb.
> The config commands are:
> gdb_memory_map enable
> gdb_flash_program enable
>
> You need to make sure you have a valid openocd flash config and for speed a
> valid working area.
>
> and if any commands need running before flashing, ie. unprotect/flash
> config then use
> target_script 0 gdb_program_config gdbprogram.ocr
>
> It is all in the info docs about gdb and flash programming.
> Also have a look here for more info:
> http://forum.sparkfun.com/viewtopic.php?t=9215&highlight=gdbprogramconfig
>
> then when you issue a load gdb will erase/program your flash.
> to verify you can use the gdb command compare-sections
>
> Hope this helps
>
