This patch feeds logged output to the output of the currently
running telnet or GDB monitor command.

This ties the action from the user tightly with the result, without
having to hunt down things in the log. Makes it much easier
to see what's going on.

Objections?

Commit after testing?


E.g. if I type:

(gdb) monitor reset init
=> all logged output here
(gdb)
