Dominic Rath wrote:
> Sad as this may be, flash is based on sectors, which can only be earsed as a whole. Erasing based on a bank and a range of sectors that should be erased (either by first and last, or by first and length) is the most natural interface to this concept.
>   
Yes and no. I have boards where different flash roms with different 
sector layouts (but same overall size) can be used. Now when erasing a 
part of flash, I need to know about sector borders, but I do not care 
about sector *numbers* - I just want to erase an address range, and I 
want to be able to specify that range in a script in a way that is 
independant of the flash used.

IMHO, the code should allow both - specifying sector numbers or 
addresses. Checking that the addresses are on sector boundaries is fine.
