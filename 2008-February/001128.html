I suggest native:
"
native

     This causes the file to contain the EOL markers that are native to
the operating system on which Subversion was run. In other words, if a
user on a Windows machine checks out a working copy that contains a file
with an svn:eol-style property set to native, that file will contain
CRLF EOL markers. A Unix user checking out a working copy which contains
the same file will see LF EOL markers in his copy of the file.

     Note that Subversion will actually store the file in the repository
using normalized LF EOL markers regardless of the operating system. This
is basically transparent to the user, though.

"

It allows everyone to use their local eol style. I hate not being able
to open the files in notepad for a quick fix.


Google hits for
svn:eol-style=native   3420
svn:eol-style=lf  242

I conclude all the cool kids use native. ;)
