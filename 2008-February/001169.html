On Wed, Feb 27, 2008 at 07:29:25PM +0100, Øyvind Harboe wrote:
> I'm wondering if it would be better to retire all those parts
> of OpenOCD documentation that are hopelessly out of
> date:
> 
> - strip away hopelessly obsolete items on web page

IMO yes.


> - go for openocd.pdf instead of a wiki. OpenOCD is *very*
>   much svn driven. wiki will always be the odd child.

The wiki is fine and useful for various things, but probably not
as the definitive documentation of openocd commands etc.
It's more useful for HOWTOs, tips etc. I'd say.

PDF doesn't sound like a good idea, IMHO. I'd prefer something
which works without extra hassle (so no HTML, no PDF). Maybe
plain text file(s) and/or the manpage/texinfo for Unix-ish
systems.


> - leave pointer in ChangeLog file to svn log
> - ditto for NEWS file

I'd rather drop them completely, and mention svn log in the README.


> A lot of trust & time can be wasted by people new to
> OpenOCD looking at stale info.

Yes, definately. There should be only one (_the_) source of
documentation, which must be regularly updated.

The texinfo sounds like a good candidate, AFAIK you can even produce
various output formats out of that (info, html, pdf).
