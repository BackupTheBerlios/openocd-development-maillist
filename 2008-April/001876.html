>
>  If two images are to be written to the flash (e.g. bootloader and main

>  and also before the second one, loosing the part programmed in the first

Sounds rare. If someone would want to write a patch to handle the above, then
I don't think I'd have a problem with it.

> > > User shall be aware wtah (s)he is doing.

>  Auto erase wipes whole sector even if just a single byte is written to the

Then you can use auto_erase=off. I don't want to tell you what to do,
but the above is no argument for why the default should be one way
or the other.

>  User shall be aware that auto erase is enabled and what it does.

>  This point seems probably most important to me.

I just don't believe the GDB load+flash feature will be used if
it isn't enabled by default.
