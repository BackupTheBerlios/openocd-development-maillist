Hi,

Attached you can find a patch for tests results ran on SAM7S, using the 
test cases in the testcases.html file. This is more to exemplify the 
testing procedure. If you are having more test cases, please feel free 
to attach them to the testcases file, as this file is just at the first 
version.
The purpose of this kind of detail is to catch even the smallest problem 
that might appear (if the test cases are designed to cover the maximum 
surface of code and functionality) and to have regression tests at the 
same time. You can easily say "testcase DBG007 worked on R607 as noted 
in the test run for that revision, not working on R628, failing with the 
message noted in the test run for revision 628".
