- reset of the JTAG controller happens *before* target asserts. This,
hopefully, should
make it possible to get ARM926ejs to work while still supporting power cycle +
reset.
- The LPC2148 target script may not need the jtag_reset statements as jtag
chain validation will fall back to a reset if validation fails, though I need to
do more testing here.
- JTAG enumeration/validation now happens when running "reset"
before targets assert reset.
