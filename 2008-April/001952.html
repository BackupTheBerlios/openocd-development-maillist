> 3. if fast fails, he can either give up or tinker with

Well, define "fail". If a DCC transfer is outrun there's data corruption, and currently no way to detect that. Fast memory writes are less critical, because the handshaking signals are evaluated after the transfer completed - it's too late to correct anything at that point, but at least we know something went wrong.

The user should be informed about the implications, and some minimum requirements like: PLL enabled, core running at x times jtag clock, external memory configured for minimum required waitstates.
