> OpenOCD stores state in global variables that is set up during startup &
> modified by commands.

That is true, and that is very important in multi-client environment.

But what I am in the need of is just this:

- OpenOCD as is
- no gdb server
- telnet replaced by a single fixed console with stdin/stdout/stderr

Surely, this would only be a simplification? The point is to toss the 
telnet and use the same interface directly. This way OpenOCD would 
become much easier to use from external scripts.

So, what I want is that when I invoke OpenOCD, I can then send all 
commands (and get all responses) from the same tty as the OpenOCD is 
running. As happens with all sorts of command line tools.
