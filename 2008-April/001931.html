chromy at asix.cz wrote:
> This is for users convenience, some MCUs contain flash compound of  
> erase sectors of different size (e.g. LPC2xxx) and it is really  
> inconvenient to erase memory based on sector numbers and write its  
> content to a place specified by address just after that - really  
> confusing.
>   
ACK. Also, we have boards where different flashs with different sector 
layout, but the save overall size may be used - it is a PITA to look up 
sector numbers for each board, so specifying erase areas by address is 
the only way to make operations scriptable (the flash layout is so that 
it fits the flash with the biggest sectors, so all operations are 
sector-aligned in all cases).
