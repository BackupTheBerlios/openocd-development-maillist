Comments?

Changes to startup/reset:

- perfrom trst/tms *before* targets assert reset
- targets should no longer assert trst/tms if they can
avoid it, updated arm7/9
- chain validation happens upon reset before targets
are assert reset

Next:

- I'll see if I can remove the requirement that the JTAG
chain is enumerated and validated upon startup
for arm7/9. For LPC2148 a reset needs to happen before
enumeration. The idea is to migrate jtag initialisation
to reset.

Also:

- added "init" command
- daemon_startup rewritten to simply issue a reset
command
