Hello Øyvind,

Here is the patch against trunk containing on-chip-loader flash driver,
someone might be interested. Its aim is similar to ecosflash driver -
semi-hosted flashing using an image loaded to the chip.
The difference is that OCL driver expects code to be running
(e.g. loaded by a script) it does not write to RAM itself.
All communication goes through DCC.

Feel free to commit, thank you.
