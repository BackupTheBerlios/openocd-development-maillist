> Any objections to these changes conceptually?
> 
> If not I'll commit when it passes testing.
> 
> I wrote up this patch and it compiles, but I haven't tested 
> it. Testing, as always, greatly appreciated! :-)
> 
> Thanks to Pavel for coming up with the concept!
> 
> - add "init" command. Executing this from within a startup 
> script will switch from config to normal execution. Issuing 
> "init" a second time is a no-op.
> - retired "daemon_startup".
> 
> Example:
> 
> 1. openocd -f target/foo.cfg -c init -c "reset run"
> 
> => run target
> 
> 2. openocd -f target/foo.cfg -c init -c reset -c "flash 
> write_image c:\xxx\foo.elf"
> 
> => write a file to flash
> 

I am ok with this, seems to work fine with the basic tests i have run.

Only issue i have noticed (not really related) is that because the server
shutdown command returns a ERROR_COMMAND_CLOSE_CONNECTION, this is treated
as a error by the find_and_run_command.
Should we also be checking the result in the parse_config_file function, as
any error is lost here.
