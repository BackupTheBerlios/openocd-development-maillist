My hope is that the user won't have to distinguish much between
C and .cfg files: they are files which are shipped in working and
tested condition with OpenOCD.

Therefore it matters for OpenOCD developers w.r.t. having maintainable
code that this issue gets resolved in the C part of the code and
that the .cfg files can be free of hacks.

Also, long term, it is good for users to be able to write their own
target .cfg files.
