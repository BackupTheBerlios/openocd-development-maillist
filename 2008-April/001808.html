Does anyone have any fundamental objections against
bringing up the telnet & gdb servers before jtag chain
examination/validation?

This is a nice feature that OpenOCD is missing.

Some advantages:

- it allows starting OpenOCD without powering up the
target
- some targets need special love and care (LPC2148 comes
to mind) to wake up the JTAG controller. This can now be
done remotely.
- other JTAG debuggers can do this flawlessly so this is
an expected feature by many developers

The nastiest problem I know of is that target init talks to embedded
ice to find out the version number before gdb/telnet servers
are launched.

My plan is to split target init into two stages.

1. get the C structures up

2. talk to the target after jtag chain examine/validation is complete and
finish init

3. (2) would happen upon every reset
