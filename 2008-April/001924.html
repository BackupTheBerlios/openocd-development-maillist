We should avoid using any commands that could be interpreted as synonyms 
of other commands. E.g., we should not have commands named both "write" 
and "program". We *could* attach meanings to these, but it's frustrating 
to new users to have to learn our arbitrarily-attached meanings to words 
they already "know".

There are only two actual operations on FLASH memory, and so I think the 
only two verbs we should use are "erase" and "write". We simply need to 
remind new users that flash must be erased before it can be written.

In that spirit, I'd suggest that the helper function that automatically 
erases and writes an image should be called:

"erase_and_write myimage.out"

(While we're at it, I might also suggest making the write command a bit 
more general and getting rid of write_bank. I'm imagining an all-in-one 
"write" command of the form:

    flash write <file> [fileoffset] [destination-address]

I'd also get rid of "erase_address", since devices don't support 
arbitrary erasure at an address granularity. (If this command currently 
does a RMW cycle, then the command should be more transparently named... )
