Just been looking at the server changes and don't think they are correct -
unless i am mistaken.
For a server the SO_SNDBUF, SO_RCVBUF and TCP_MAXSEG should be set before
the call to listen - they are then inherited by the client.

win32 supports all these changes except TCP_MAXSEG so can also benefit
especially from the TCP_NODELAY option.
we can also use the check_pending function in win32 aswell.
