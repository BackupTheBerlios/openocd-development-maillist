Two things:

1. you need to add the "init" command before you can execute e.g. "wait_halt".
BTW "wait_halt" is pretty much obsolete now that "halt" is synchronous.

2. If you  post the command line options and the actual config files used,
that would help. Also CPU type and eval board would be helpful to know.


3. It looks like an electrical problem additionally. The LPC2148, for
instance, needs
a pulldown on the RCLK upon reset for the JTAG controller to be enabled.
Without this pulldown the JTAG pins are used for GPIO. The LPC2148
is "clever" in this way as it manages to dual-purpose the JTAG/GPIO
pins. This means that your JTAG interface could work just fine with
Xilinx and fail with a CPU. The JTAG standard does not have a carved in stone
defined electrical interface like, e.g. serial ports or USB.
