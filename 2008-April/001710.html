On Sat, Apr 5, 2008 at 10:49 AM, Spen <spen at spen-soft.co.uk> wrote:
> >
>  > - add "init" command. Executing this from within a startup
>  > script will switch from config to normal execution. Issuing
>  > "init" a second time is a no-op.
>  > - retired "daemon_startup".
>  >
>
>  One thing i forgot to mention, not keen on removing the daemon_startup.
>  For people who like to do everything in a script, this changes the behaviour
>  - forcing them to use the command line.

As Pavel explained, "init" can be used from a script as well.

"daemon_startup reset" is the equvalent of issuing these two commands:

init
reset

I need to read up on Dominic's posting, but if the above is not
equivalent to daemon_startup reset, then that is problem with
reset that needs to be addressed.

It is of course possible to keep "daemon_startup reset" as deprecated
and just an alternative syntax to the above.

I would like to point the user towards the more flexible capabilities
of "init" + cmd's as he may well want more than just to reset the
target.
