My knee jerk reaction is that OpenOCD should add entries
to the XML file for the rest of the memory to be read/write.

I can't see a reason why we would want any other default.

For the user to tell OpenOCD where e.g. his registers are
in memory is just make-work.

The story might be a bit more subtle though. I've read
a bit about GDB target memory caching and I did
take it for a spint without much luck. If target memory
caching works in GDB, then GDB would need to know
the difference between RAM and uncachable read/write
memory.
