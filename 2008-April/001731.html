Hi,

Below patch mainly updates cortex_m3 debugging, only tested on stm32.

- single core context used, removed debug context as thought unnecessary.
- DCRDR now used to access special core registers - info is currently
omitted from the cortex_m3 TRM
  ARM have told me this is the preferred access method and the docs will be
updated soon.
- now checks for User Thread Mode and Thread mode when halted.
- removed repeated function declarations from command.c

If everyone is ok i will commit.
It would be nice to have it double-checked on luminary, but i have no access
to silicon. I see no issues however.

Cheers
Spen
