Hi

I think the auto_erase should NOT be on by default. For low level work 
with a informed user in control
flash programming consists of erasing + programming, except on some 
targets like AT91SAM7 that handles this for us,   or if we
erase a big chunk and do incremental writing.
If we set up OpenOCD to be driven from another program like GDB the we 
must configure OpenOCD accordingly and
differently depending on the target flash setup.  Config scripts will be 
necessary for different usage scenarios, and gdb is only one scenario 
but an important one. My personal feeling is that default should be not 
to auto_erase.

Of course any behaviour works as long as it can be turned on and/or off.

Regards,
