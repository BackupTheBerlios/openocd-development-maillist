Dominic Rath wrote:
> -------- Original-Nachricht --------
>
>   
>> - the reset mode parameter is now DEPRECATED. It is implemented
>> as an optional parameter with default reset_init. This is to streamline
>> things w.r.t. the target library.
>>     
>
> Why? Optional maybe, but definitely not deprecated.
>
>   
Since some (eg. Zylin) may use OpenOCD as an API, the "reset" command 
had different meanings, depending on the script, so the outside 
application got confused. This is a step to nail down the default action 
of reset, and mandatory to type the action of reset when invoking the 
command if something else is desired.
