Spen wrote:
>>> any objections to that making auto_erase default as well then?
>>>   
>> No - having auto_erase on sounds good IMHO.
>>
> No sounds fine, only tweak needed would be to disable while gdb is
> programming otherwise the flash will get erased twice.

On contrary, I do not consider this to be a good idea.
Using flash write multiple times on areas which fall into the same erase sector
would not work then. User shall be aware wtah (s)he is doing.
Second, some parts do feature erase before write on their own in the hardware,
thus auto_erase is of no use here, only the FLASH gets worn sooner.

I believe this was discussed before, please see
https://lists.berlios.de/pipermail/openocd-development/2007-September/000355.html

I would rather keep auto_erase disabled by default.
