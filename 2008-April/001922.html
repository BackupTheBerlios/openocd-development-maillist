Jonathan Larmour wrote:
> Out of interest, some programmers do a blank check first. Without a 
> software agent running on the target (as many other JTAG programmers 
> use) that's more difficult for OpenOCD as it would require reading all 
> the target area, but you can use a blank check heuristic - check the 
> first, say, 16 bytes and last 16 bytes of the flash to be programmed, 
> and also for the start/end of any sectors in the middle.
OpenOCD has code to do a full erase check - either using code on the 
target (if a target buffer is available), or (slow) by reading the whole 
area from the host and comparing with 0xFF.
