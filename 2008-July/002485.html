I'm considering changing .cfg files to tcl.

This *DOES NOT* involve any syntax changes!

This *DOES NOT* involve any behavioural changes *if we do not want to*.

The main issue is with exceptions/errors:

- If a tcl exception occurs, tcl will terminate w/an error message.
Bonus: file name & line number is displayed. This would only happen if
tcl code was added to the .cfg file, so this is not a change to
old .cfg file behaviour
- If an OpenOCD command fails, then we have a choice about
what we do by default. We can throw a Tcl exception.
This will abort the script and show an error message w/line
number. Alternatively we can continue execution as today.
To get a tcl exception thrown for a failed OpenOCD command,
add the "openocd_throw" prefix.


My preference is to throw a Tcl exception by default upon
a failed OpenOCD command.

Motivation:

- explaining .cfg files becomes easier: they are Jim Tcl files
- simplify OpenOCD internal code.
- promote scripting
