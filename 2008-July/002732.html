Would you be interested in porting target_process_reset to Tcl?

I've cleaned up a few things in target_process_reset().

Basically we'll need a tcl implementation that is backwards compatible
but overrideable by the target configuration script.

This should allow for more flexible handling of e.g. heteregenous
JTAG chains(non-ARM + ARM devices mixed).

The events are there because this is what users are familiar with.

You'll need to implement low level commands that the tcl
implementation of reset uses.
