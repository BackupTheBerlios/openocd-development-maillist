I skimmed through it and much of it is along the lines of
what I've been thinking.

I'd like to toss in more backwards compatibility though.

My thinking is that we break down the current reset command
into lower level commands used by a default implementation
of the reset command.

The default implementation of the reset command is backwards
comptabile, but can be replaced by the target script.

The patch I posted yesterday was a step in this direction in
demystifying parts of reset. E.g. targets no longer have
a concept of reset_mode from the reset command.
