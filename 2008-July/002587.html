There is one major change/bugfix to look out for:

if there is an error in a script, then the script is now aborted. An
error message is shown w/file & line number where execution
was halted.

This change was intentional and the choice was
made this way to be more consistent w/error message
& exceptions.

Of course there is a compatibility cost, but long term I'm convinced
that this was the right decision.

Is this what you are seeing?
