Hallo List,

please do not remove stable functionality. What I mean
is, it is OK if new functionality will be added. But
if the new functionality makes the same as the old
way, please let the old way in. The old way can be removed
if the new one is stable and the documentation and examples
was changed.

A good example is the daemon_startup feature. In the moment
r834, r835 this functionality is removed and you must use an
other way to get the same functionality. This will create
a lot of support in the future.

https://lists.berlios.de/pipermail/openocd-development/2008-July/002674.html

The problem is that the user will use the latest trunk
of openocd. They do not know if this is a stable version or
in development. But they assumed to use it with the "old"
functionality.

Best regards,

Michael
