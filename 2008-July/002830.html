Whether a ocd_flash_banks belongs in flash.c or in tclapi.c would be
a matter of taste, I guess.

Kinda like choosing between sorting your CD's on artist or album
name... Personally I sort my CD's by color, which makes it
all look neat and orderly, but it is indistinguishable from complete
chaos when trying to find anything. :-)

It will make the tcl stuff less incestious with the rest of the code
and force us to think a bit about the API to e.g. flash/target/jtag
from other code.

The plan is to mess around with Tcl, so if we move it now,
we'll see fewer version # bumps all over the place.

Once we write the documentation :-) it will be a tad
easier to find all tcl API procs.


Regarding the patch: I committed the tclapi.c file so as to reduce
patch size, but it is not currently in use. If we do not apply the
rest of the patch, then tclapi.c should be deleted.
