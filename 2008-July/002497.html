No. People can generate whatever they want. If I can handle both sides 
of the connection (TCL generating output, language X accepting it), it 
is just fine. There are so many script languages in use that I think 
this TCL-based approach has its merits. It is a practical impossibility 
to produce a usable interface for all purposes and languages.

However, what I am missing is a single example for *any* language. How 
to use port 5555 in practice. Even a single item of data passed from the 
target system to the user script would suffice for this purpose.

I know Öyvind's approach has some downsides, no doubt about that. But it 
is script-language -independent, future-proof, and maintainable. Another 
possibility would be to make a RPC system which would then double the 
maintenance burden.
