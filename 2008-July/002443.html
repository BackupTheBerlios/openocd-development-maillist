A *scripting* friendly library. Somewhat less friendly to C/C++. Anything
but C/C++ would probably be happier with this interface than a C/C++
friendly interface. E.g. Java can easily talk to a socket + strings, but
adding JNI to the mix gets annoying.

I've always considered Telnet a user interface and here that userinterface.

A fair amount of the Telnet UI code would be rewritten to small tcl'lets
here.
