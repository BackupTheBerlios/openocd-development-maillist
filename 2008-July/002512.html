There is always an "over the wire protocol" you can dance around the issue all you want.

As you say: 

>> The client *decides* what code to run on the server(e.g. on 
>> the OpenOCD command line + injects scriptlets, etc.)

Thus the defined over the wire protocol is:
	(A) the invoking of existing scriptlets functions.
	(B) the ablity to create new "scriptlet functions".
	(C) the output of scriptlets is TCL list (ascii) format.

Others [I believe accidentally replying privately] have suggested we might look at JSON.

	http://www.json.org/

It too has a "name/value" pairs
