 >> One outstanding issue is to be able to override the handling of 
openocd commands in config scripts. This capability was lost when 
"openocd_throw" was retired.

I believe this to be simple.

For all C functions - that are not meant to be used directly by the user 
- they have a "_C_" prefix.
A polite "human friendly wrapper" is then created.

Perhaps - Even those that are meant to be used by the user should have a 
_C_ prefix.

Then - dynamically create a  TCL proc like this:

       proc  foo { args }  {   _C_foo [set args] }

I am not sure if Jim/TCL supports the magic "args" parameter.  Anyway - 
this would mean some tweaking on the existing C functions.

See:  "Variable Numbers of Parameters" here

    http://www2.lib.uchicago.edu/~keith//tcl-course/topics/procedures.html
