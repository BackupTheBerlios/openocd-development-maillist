The ft2232 driver in OpenOCD puts the chip in MSSP mode and sends  
packets to toggle the various JTAG signals.  In a code download, the  
data will be a series of IRSCAN and DRSCAN commands in rapid  
succession.  Specifically, it will be writes to the device's memory  
via the target's debug interface.  The ft2232 driver will attempt to  
batch the various commands to be sent into a bulk transfer.  The error  
indicates that the bulk write to the chip failed for some reason.  You  
might try an example libftdi program that does MSSP bulk transfers.
