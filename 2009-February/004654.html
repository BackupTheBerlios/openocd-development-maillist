There are at least 3 ways to move forward:


1)  http://www.nabble.com/snooping-the-usb-packets-td14684534.html


2) libusb has its own debug capability.   I believe you only have to set 
an environment variable named USB_DEBUG to turn it on.  Then run your 
app after that.  See
https://www.codeblog.org/viewsrc/libusb-0.1.10a/usb.c
and about line 192.


3) You can enable debugging within OpenOCD by defining _DEBUG_JTAG_IO_ 
on the command line to the compiler.   Something like this:

$ make clean
$ make "CFLAGS=-D_DEBUG_JTAG_IO_=1"

this will create OpenOCD with extra debugging options in it.  I think 
you still have to turn them on with debug level 3 however in your 
configuration file.  The debug options are richer with _DEBUG_JTAG_IO_, 
but they are not turned on if I recall, until you up the debug level in 
the runtime configuration.
