On Jan 31, 2009, at 9:14 AM, Dick Hollenbeck wrote:

> This is a Doxygen configuration file for the project.  If it is put  
> into the root of the project, and from there the doxygen command is  
> run, the doxygen command will use Doxyfile as its default  
> configuration file.  The source code documentation for the project  
> will then be put into the doxygen directory as *.html files.  This  
> may be helpful for new developers trying to get a quicker  
> understanding.
>
> Source code comments catering towards Doxygen can be used to  
> embellish the Doxygen output at little cost to a developer.  See  
> function
>
> /**
> * Function ft2232_stableclocks
> * will send out \a num_cycles on the TCK line while the TAP(s)
> * are in a stable state.  Calling code must ensure that current  
> state is
> * stable, that verification is not done in here.
> * @param num_cycles is the count of clocks cycles to send.
> * @return int - ERROR_OK or ERROR_JTAG_QUEUE_FAILED
> */
> static int ft2232_stableclocks(int num_cycles, jtag_command_t* cmd);
>
> as an example.
>
>
> Only this configuration file is needed in the repository.   The  
> *.html files can be generated by a developer at any time with a  
> recent version of Doxygen and do not need to be part of the SVN  
> repository.
>
>
> You will see Doxygen comments on some of the new work I have been  
> doing.
>
> When committing, please remember to set the svn-eol style to native.
>
>
> Hope this helps,
>
> Dick
>
