I've been working on a project (http://openpilot.org) that is using a
JTAG based on the FLOSS-JTAG.  When I tried using two JTAGs at the same
time, I found that openocd couldn't tell them apart because these JTAGs
didn't have serial numbers.  The USB descriptors on these devices are
absolutely identical so none of the existing differentiation mechanisms
will solve the problem.

This patch provides a way to use their unique usb bus address to 
distinguish them from one another.  I've been using it successfully
for a few days.  This concept is probably generalizable for other
inexpensive USB JTAGs without serial numbers but I've only got FT2232
based ones to work with so this patch only applies to FT2232 devices.

Comments appreciated.

Stacey.
