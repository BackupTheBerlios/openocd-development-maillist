What you'll need to do is to rename the local variable
"wait" to "wait_var" or something.

The problem is that there is a global variable with the
same name as the local variable, which -Wshadow will
warn you about.

Once you have fixed this particular warning(almost certainly
trivial search and replace job), then there may be a few other
warnings for that OS/GCC mix.

And submit a patch obviously :-)
