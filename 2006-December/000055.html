Hello,

I want to create a OpenOCD hardware JTAG key, without using the FTDI2232.
Therefore I have some question about the "TAP controller" in OpenOCD.

Take a look in bitbang.c, here the function bitbang_state_move can change
the TAP state.
The TAP_MOVE macro decodes the tms_scan bits, which will be used for the
next 8 clocks.

TAP_MOVE use the following macro:

#define TAP_MOVE(from, to) tap_move[tap_move_map[from]][tap_move_map[to]]

"from" and "to" in the range of tap_state, here from 0x0 to 0xf.

But the tap_move_map has some negative values, what happen if a negative
value is
used as an index for tap_move? Or is this an invalid state transition?

For the moment I give up to implement OpenOCD over ethernet. Therefore I
want to create
a USB solution with a CPLD for the JTAG part. The USB will be handled by an
STR710 and
the CPLD is memory mapped.
