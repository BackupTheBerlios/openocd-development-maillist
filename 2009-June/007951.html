It's still not *quite* working as expected, since talking to
the EmbeddedICE module on the ARM is giving nonsense, but at
least I seem to be able to do things without explosions.

Following are three patches that improve handling of disabled
targets, and the process of enabling them:

 - Don't let background polling interfere ... there were
   two C bugs here, and one English one (spell "continuous")

 - After enabling a TAP, examine any targets it hosts.  They
   were ignored on server startup.  Don't ignore failures...

 - "jtag newtap ... -disable" reflects state after TLR, so
   update the docs and make the JTAG reset callback update
   that state.  (Plus fix some goofiness there.)

I don't think any of this will affect TAPs that aren't
initially declared as disabled.
