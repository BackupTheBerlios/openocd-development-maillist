On Sat, Jun 20, 2009 at 10:53 PM, Thomas A. Moulton<tom at moulton.us> wrote:
> If libusb and libftdi works, then please build it and make it available
> so it can be used.

I believe it can work under the platforms where libusb-win32 device
driver works (Win 2K, Windows XP 32/64bit, Vista 32bit). It is
not so simple for normal users but I think it can be made to work.
The remaining issue is Vista 64bit which seems to gain popularity
now. That can be solved as well if some company can sponsor the
digital signing of the core libusb-win32 driver. The hardware vendors
can also choose to WHQL their particular device (specific VID/PID)
if they want to differentiate from others and gain some competitive
advantages.

But there are other issues with this approach, the serial port
seen in some JTAG tools will be gone due to the use of libusb-win32
driver (as the driver for the whole USB composite device).

The performance issue may not be the most critical one. But
the losing features and support issues (the user may not be able
to install libusb-win32 driver properly) can be the killing factor
for the hardware vendors.
