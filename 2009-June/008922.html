Hi,

I had a problem with the olimex stm_h103 board using the orginal
config files from SVN r2405. The problem was, that in the
target/stm32.cfg the work area size is set to 16k which is good for
the smallest stm32 but not for the STM32103RB found on the olimex
board which has 128k.

I have created a small patch, that introduces the variable
WORKAREASIZE to target/stm32.cfg, so the user can easily set the
maximum flash size for his target. I have also added the variable with
the correct value to the board/olimex_stm32_h103.cfg.

This solution will also help Gene Smith with his problem reported on
Wed, 29 Apr 2009
(https://lists.berlios.de/pipermail/openocd-development/2009-April/005919.html)

Regards

Kevin
