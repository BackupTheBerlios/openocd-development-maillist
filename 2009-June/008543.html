Also see a bit later in that same file:

 http://www.gnu.org/licenses/gpl-faq.html#GPLIncompatibleAlone

Which includes:

  If a program P is released under the GPL that means *any and every
  part of it* can be used under the GPL. If you integrate module Q,
  and release the combined program P+Q under the GPL, that means any
  part of P+Q can be used under the GPL. One part of P+Q is Q. So
  releasing P+Q under the GPL says that Q - any part of it - can be
  used under the GPL. Putting it in other words, a user who obtains P+Q
  under the GPL can delete P, so that just Q remains, still under the GPL.

  If the license of module Q permits you to give permission for that,
  then it is GPL-compatible. Otherwise, it is not GPL-compatible.

For "P" read "OpenOCD".  For "Q" read FTDI's D2XX library.  Quite
obviously, FTDI's code is not GPL-compatible; no source available
is just the most obvious issue.  Q.E.D.
