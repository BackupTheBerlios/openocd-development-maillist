Hi,

I remember there was a configure option to enable/disable rtos in the
first version of the rtos patches. But when it was committed, this
configure option was removed. I'm currently considering using
multi-thread model for debugging multi-core processor, for example
dual-core BF561. This will conflict with rtos, which also uses
threads. So I'm considering adding back the rtos enabling configure
option. I will also add a new option to enable/disable multi-thread
for multi-core and make these two options exclusive to each other. Do
you think this is a good plan?

Regards,
