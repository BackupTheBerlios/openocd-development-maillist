I missed answering some parts of your email...

Making an option to turn it off should be simple, but is not currently implemented - I agree this should be done.

The standard GDB thread commands can be employed to use it. There are no special commands required.  The thread awareness is allows visibility of threads, stacks and locals for each thread, in the same way that native debugging does. If you use eclipse, or similar GUI front end, then the commands for retrieving threads and stacks are all automated, requiring only the "-rtos auto" config command.
