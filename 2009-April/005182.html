On Apr 2, 2009, at 3:36 AM, SimonQian wrote:

> Hi,
> I noticed there is a thread about supporting other
> programming interfaces except JTAG.
> How is it going?
>

There was some talk, but no action.  It would require quite a bit of  
infrastructure work to remove the assumption that every pld and target  
is attached via JTAG.  SWJ seems to be most requested alternate  
interface, but I'd like to see the OpenOCD internals move to a  
interface-neutral design.  I have ideas of how to make this happen,  
but time is currently not something I have any spare of.  I _might_ be  
able to get to this sometime in July.

> I'll release a program with these support.
> I tested it on AVR_ISP/JTAG, C51_ISP, C8051F_C2/JTAG,
> PSOC_ISSP, MSP430_JTAG(non-standard JTAG), and even
> STM32_ISP(Comm port).
>
> If such support is actually planed, I'll try to implement
> these support in OpenOCD.
