I went ahead and did some moving of some files relating to the 
AT91SAM9260 and the Olimex SAM9-L9260 development board.  I also created 
another file for the Olimex JTAG TINY that doesn't include the 'A'.  
Apparently, the description for the JTAG tiny can differ between Windows 
and Linux Operating Systems and this can make a difference, so I figured 
it might be nice to have two of them.  Probably could do an include to 
reduce redundancy, but I'll leave that decision to some one else.

Partial annoyance:  the created patch file lists some files as having a 
new property set called 'svn:mergeinfo', but their contents are not 
included.  Here is the 'svn status' on modified rev1333

    R  +   src/target/target/at91sam9260.cfg
    D      src/target/target/at91sam9260minimal.cfg
    A      src/target/board/olimex_sam9_l9260.cfg
    A  +   src/target/board/unknown-board-atmel-at91sam9260.cfg
    A      src/target/interface/olimex-jtag-tiny.cfg

Here is the general idea because the two new files are not fully listed 
in the patch:
    target/at91sam9260.cfgR1333 -> board/unknown-board-atmel-at91sam9260.cfg
    target/at91sam9260minimal.cfgR1333 -> target/at91sam9260.cfg
