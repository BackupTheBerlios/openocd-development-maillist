Please DO NOT remove the in_check and in_handle from JTAG layer.

Checking directly in the JTAG layer is an wonderful feature for the 
OpenOCD JTAG server. It could accelerate a lot the JTAG interface 
version 'Stream'.

Retiring these fields will only make the code smaller, but will not 
accelerate the server itself.

Also, please keep these fields.
