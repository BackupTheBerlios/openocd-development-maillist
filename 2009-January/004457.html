I have one concern regarding the configuration files for the 
AT91SAM9260.  There's an AT91SAM9260minimal.cfg that should be 
AT91SAM9260.cfg and the current AT91SAM9260.cfg should be a board file.  
I submitted a patch to the list to fix this the other night, but it 
might be worth having it added to this release.

Tested build on Fedora 10 using following commands:

libftdi:
    ./configure --enable-parport --enable-parport_ppdev 
--enable-ft2232_libftdi --enable-ep93xx --enable-at91rm9200 
--enable-usbprog --enable-presto_libftdi --enable-jlink --enable-vsllink 
--enable-rlink --enable-dummy --enable-gw16012 --enable-amtjtagaccel 
--prefix=$HOME/tools/openocd-0.1.0/
    make all install

libftd2xx:
    ./configure --enable-parport --enable-parport_ppdev 
--enable-ft2232_ftd2xx --enable-ep93xx --enable-at91rm9200 
--enable-usbprog --enable-presto_libftdi --enable-jlink --enable-vsllink 
--enable-rlink --enable-dummy --enable-gw16012 --enable-amtjtagaccel 
--prefix=$HOME/tools/openocd-0.1.0/
    make all install

Both have successful build and run, but I am only able to verify that 
the libftdi version works to some extent with my JTAG device and board.  
I'm having issues with the ftd2xx driver, so it isn't OpenOCD.  Man 
pages appear to be in place.
