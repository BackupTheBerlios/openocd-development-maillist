Dean Glazeski pisze:
> In an effort to support Freddie Chopin, who is trying to put together 
> the Windows binary for OpenOCD, I have been trying to get OpenOCD to 
> just compile in Windows.  I've tried following some instructions at 
> SparkFun <http://forum.sparkfun.com/viewtopic.php?t=11221>, but to no 
> avail.  I have tried just doing the configure script in Cygwin and then 
> running make from command.exe.  No go there too.  I keep getting caught 
> on a missing usb.h when make hits usbprog.c.  I just can't seem to shake 
> it.  Can anyone provide some good direction for compiling in Windows?  
> Is it in the documentation for OpenOCD?  Any help is much appreciated.

i think that the documentation of the project is not precise about that 
- probably most of the developers have Linux based systems...

anyway - I've compiled OpenOCD 0.1.0 using MSYS, not cygwin, but the 
solution is probably the same - libusb library. i had to download the 
package from the project website, and put usb.h in the ...\mingw\include 
folder

my configure options were:
--enable-parport --enable-parport_giveio --enable-ft2232_ftd2xx
--enable-ep93xx --enable-at91rm9200 --enable-usbprog
--enable-presto_ftd2xx --enable-jlink --enable-vsllink --enable-rlink
--enable-dummy -enable-gw16012 --enable-amtjtagaccel
--with-ftd2xx-win32-zipdir=[path to ftd2xx]

4\/3!!
