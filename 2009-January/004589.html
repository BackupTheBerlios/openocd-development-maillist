Currently openocd does not have a bug database and I think this
is a reasonable way of doing things. For now.

Short story: post a reminder to the list if something is dropped
on the floor.

These reminders are welcome and will not be considered nagging!

If a well formulated patch is not applied within a week, then it has
been accidentally dropped or maintainers are off somewhere on
a short leave :-)


Here is how things are effectively handled:

- a bug is described and discussed
- either a patched is formulated or it is not. If no solution is found
then the issue is not tracked in a database, but anyone is free
to post reminders or new info.
- any reasonable patch that moves the project forward is, most times,
applied immediately(a couple of days).

So what about stuff that gets dropped on the floor?

- if no solution exists or resources are not available to solve
the underlying problems, then "dropping" the issue on
the floor is fine.
- if there is a patch that should have been applied but is
dropped, then we rely on the community to post a reminder


OpenOCD effectively drop stuff on the floor that there are no
resources to follow up on or nobody cares (enough) about.
