> Just for your information, we released libusb-win32 1.2.1.0
> a week ago.
>
> With this release, the inf-wizard can automatically install
> the device driver. So users may not need to go through the
> manual process of installing libusb-win32 device drivers.
> It has also embedded the necessary binaries. Therefore the
> only thing you need is to give the user the inf-wizard.exe file
> and the user can go through the wizard to install the device driver.
>
> An alternative is Zadig/libwdi. Zadig supports both libusb-win32
> and WinUSB.
> http://www.libusb.org/wiki/windows_backend
>
> Same as 1.2.0.0, the kernel driver libusb0.sys in 12.1.0 is signed
> so that it will work under 64bit Windows OS which requires
> digital signature for kernel drivers (KMCS).
>
> V1.2.1.0 (07/28/2010)
> =======================
> * Updated bulk.c to include async i/o example code.
>
> * Fixed usb_install_driver_np() issue with inf-wizard generated infs.
>
> * Fixed inf2cat.exe issue with inf-wizard generated infs.
>
> * Added 'Install Now' feature to inf-wizard. (libwdi)
>
> * Added embedded libusb-win32 binaries to inf-wizard. (libwdi)
>
> * Added libwdi (http://www.libusb.org/wiki/libwdi) to inf-wizard.
>
> * Added get cached configuration request to usb_open(). This is a new
> control code that involves no device i/o and allows
> usb_set_configuration() to be omitted if the driver has already
> configured it.
>
> * Fixed set_configuration() failure for devices that do not properly
> support get_configuration().
