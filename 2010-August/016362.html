Presumably one of the benefits of trace would be to
be able to step back/forth.

Here is an idea on how to represent this in gdb: add a new
thread to the gdb remote protocol. This thread would represent
each instruction in the trace as a call frame in the GDB
"bt" instruction.

No change required to GDB user interfaces :-)
