Someone vote for clear and some other vote for fill. In case of clear
it is possible that we will not find an uninit variable, because some
test inside the source are tests like (x!=NULL). If we use the fill
option, the software will possible crash and perhaps this could help
us to find the place.

I think we should have a software for the "end-user" where the clear option
is on.
But for the developer which try to find bugs it could be better to use the
fill options?
