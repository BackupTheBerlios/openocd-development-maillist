I need to profile interrupts on a target.

To do this I modified OpenOCD to sample the PC and then I spit out a
gmon.out that
gprof can understand.

However, it seems like halting, sampling the PC and resuming is a bad
way to do this
because the target will remain halted for a long time. So I either get
a really small number
of samples or lots of interference with execution. Not a good thing
since I want to
profile network interrupts...

Is there a way to sample the PC while the target is running,
alternatively would it
be possible to write a custom code path to halt + sample pc + resume?
