Hello List,

I have added a small patch from Dominic for the ATMEL flash:
https://lists.berlios.de/pipermail/openocd-development/2007-December/000433.
html

and hopefully fixed the malloc problem inside "cfi_flash_bank_command".

Now the Ethernut3 board is working and the user can program the external
flash. This was broken some month before.

I do not want to commit to much changes at this step, therefore I will
added next the clear_malloc, fill_malloc function which could help us
to find some other malloc problems in case there was here.

Don't panic, I will only make small changes in the replacement file,
where the user can decide to use clear_malloc or fill_malloc. I think
this will help other to implement some new functions, and on the other
side we can use the fill_malloc function try to find some more memory
"bugs".
