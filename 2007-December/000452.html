Hello List,

if you forgotten to unprotect a sector before program it, you will
get an abort error in case of the second write command. Take a look
at the example. I have use here a LPC2294 target with a TE28F160 flash.

How to produce the error?
==========================
1: flash probe 1
2: flash write 1 test.bin 0 -> produce write error
3: flash protect 1 0 0 off
4: flash write 1 test.bin 0 -> produce abort error

How to avoiding the error?
===========================
1: flash probe 1
2: flash protect 1 0 0 off
3: flash write 1 test.bin 0

The file size of test.bin is 237 bytes, the start address of the flash 
is 0x80000000.

Is the abort error normal?
