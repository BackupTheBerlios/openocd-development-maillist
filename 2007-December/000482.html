Hello List,

I have added the patch with the new flash functions now.

What is new, what is changed?

--- Changed ---

"flash erase", DEPRECATED, use erase_sector instaed.
"flash erase_sector" had the same syntax like "flash erase" before.

"flash write", DEPRECATED, use write_binary instaed.
"flash write_binary" had the same syntax like "flash write" before.

--- New ---

flash erase_address <address> <length>
This will erase all the sectors in the address range from <address> to
<address+length>

flash verify_image <file> [offset] [type]
The same like flash write_image, but here the flash will be verified only.

The function are working, but in some case a wrong error message is
displayed.
For example "write error" in case of a verify error. But Øyvind, told me
that he is
working on this.

I have added this patch with these small "miss alignments", that the other
users can
help us to find more of this "bugs" if available. Even I could create a
windows version
on request.

Even I have added the patch from Spen, "new command an cleanup", for more
information
for this, take a look at:
https://lists.berlios.de/pipermail/openocd-development/2007-December/000476.
html

I think memory leak fixing are important.

The fileio patch was not added here, this will come in the next step.

I hope these patches will not break the functionality. And we need some help
for testing.

There are some miss alignments, for example:

- try to program the internal LPC flash at address 0x100, it is not working.
Yes, this is a limitation of the LPC itself. Perhaps we can fix it?
- or the problem with the ihex file here:
https://lists.berlios.de/pipermail/openocd-development/2007-December/000467.
html

Thanks to Øyvind and Spen for the patches.
