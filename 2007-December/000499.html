> I don't think so. Only RAM and NOR flash are memory mapped.

I think this makes sense for memory technologies that have a straightforward
memory mapped read cycle but a convoluted write/erase cycle. There are lots
of technologies that don't fit this bill obviously(e.g. NAND that is
too big to fit into the
address space, out of bounds data, etc.)

I know the difference between the technologies, I just don't care about those
differences when I'm debugging a program.  Somehow the program has to
make it into that memory with a minimum of fuzz.

Developers who are blissfully ignorant about memory technologies with
a straightforward
read cycle and a convoluted write/erase cycle should be allowed to
remain so until
there is a need for them to know about the finer points.

That's just my opinion anyway.
