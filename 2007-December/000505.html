The Insight parallel jtag cable is nothing special, just some buffers.  It's 
Jflash that does the magic.  Either Jflash_linux or Jflash for windows does 
the trick.  Came with my Lubbock dev board way back when.  Note that about 
half the programming time is taken up by the verification process- Jflash 
automatically unlocks and erases the number of blocks based on the binary 
file size, programs the flash, then reads the flash back to verify the 
programming.  A 438K bin file takes less than 5 minutes start to finish.  A 
MacGreggor Raven Wiggler does it in about 20 seconds.  So 29 minutes for a 
106K bin file has me thinking maybe there's an algorithm issue somewhere.

I had to fix a bug in Jflash when I started using P30 flashes as the algorithm 
for figuring out the size of the part based on the number of blocks read from 
the cfi fell over when it encountered an odd number.
