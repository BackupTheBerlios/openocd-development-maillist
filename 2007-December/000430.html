Dominic Rath wrote:
> On Monday 10 December 2007 15:51:23 Øyvind Harboe wrote:
>   
>> I would like "flash erase" to take address & length as arguments.
>>
>> This would of course break the syntax of "flash erase", which would
>> otherwise have to
>> be marked as deprecated in the help text + a more awkward name for the new
>> flash erase command.
>>
>> Any objections?
>>     
>
> I don't think it's a good idea to deprecate the existing behaviour. Personally 
> I don't want to erase based on addresses - I prefer the current way using a 
> bank and a range of blocks. Happened too often that I typed addresses with 7 
> or 9 hexdigits... ;)
>
> Is there any problem with "flash erase_address"?
>   
Yes - different flash roms that may be populated on the same board. I 
have multiple qualified flash roms with different sector layout, which 
work with the same software. That means to update the bootloader, I need 
to specify different sector numbers, although the addresses are the same 
- this is bad for scripting stuff.

> We could also deprecate "flash erase" and add a "flash erase_block", to have a 
> more consistent naming. "flash erase" would then map to "flash erase_block" 
> until we decide to a) remove it completely (preferred) or b) change it 
> to "flash erase_address".
>   
Not sure. Since you can only erase whole blocks in a flash, there is not 
much use in distibguishing these two names. No matter what the command 
is called, I would prefer a flexible syntax that lets me specify either 
a list of sectors or a range of addresses - I think there is a use for 
both methods.
