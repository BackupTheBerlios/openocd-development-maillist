Hi,

Just committed the attached patch that adds support for the restart packet
(extended gdb)
It requires that gdb is attached via target extended-remote rather than
target remote.

Then the target will be reset when a gdb 'start' command is issued.

Similar behaviour can be obtained in standard mode using the gdb_detach
reset openocd config option.
