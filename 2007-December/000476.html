Find attached a patch for review:

1. Add support for parport_write_on_exit <on|off> command
The parallel driver was tweaked a few versions ago (rev 204) to write a
known value to the parallel port on exit - this has the impact on holding
targets in reset. Sometimes this is not the required behaviour, for example
when using gdb_detach and exiting openocd.
The old behaviour is now the default - do not modify the parallel port on
exit.

2. Also changed a few global variables to NULL after a memory free command.

3. Fixed a couple of memory leaks in jtag.c (jtag_validate_chain)

Cheers
