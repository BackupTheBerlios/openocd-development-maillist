On Sunday 07 February 2010, Julien Iguchi-Cartigny wrote:
> Hello David,
> 
> > If you're using Linux, I suspect you just forgot to install
> > the UDEV rules which automatically set up access for you.
> 
> I didn't have time to investigate but it seems that the libftdi library
> requires root right. I don't know if it could be solved with UDEV rules,
> or if the driver uses very specific kernel calls.

The only reason libftdi should need root access is because
the devices' special file is protected against non-root access.
One of the tasks of udev is to set up the file mode correctly.


> > Look at Stellaris eval boards too.  There's variety in them; you
> > could get some of each, supporting different kinds of projects
> > and peripherals, while saving the cost of discrete JTAG adapters.
> > The very lowest cost boards (LM3S811 EK) are about $US 50, which
> > is less than many discrete JTAG adapters...
> 
> I don't understand: do you mean the jtag is included ?

Yes.  These boards include an ft2232 chip wired up to serve as
 (a) JTAG adapter on the A-channel, and
 (b1) uart/serial adapter, on the B-channel.

Or instead of the UART adapter, and given some software we don't
yet have, that B channel can collect SWO trace data.

That is, it does more than the JTAGkey you mentioned (which doesn't
use its B-channel) ... albeit with the same chip, integrated onto
the devel board.  (Plus, board power can come from USB too.)
