Git has an RC2 tag, and when you autoreconfig it comes out as RC2.
Source archives are downloadable from SourceForge, and its news has
a brief blurb.

  https://sourceforge.net/projects/openocd/

Berlios downloads are in the works.  (They take forever to do, since
Berlios uses such an ancient/goofy clone of the SourceForge codebase.)

We're hoping that RC2 will be brief, since RC1 has been cooking for
so long (and with relatively few problems).  As in ... if no trouble
shows up in the next week, I'll be inclined to ship v0.4.0-final by
about this time next week.

There's a known regression in "help" handling (for ARM11 and similar
commands), which is minor (workaround: use the User's Guide) and not
looking like it'll get fixed any time soon.  So I think we'll be stuck
with that regression staying unfixed; sigh.  (See the Trac ticket...)

The ARM920T cache bug looks like it may be fixed, but I've not seen
confirmation of that yet.
