> The first one looks like the correct and functional driver that uses

That is for Channel A which uses the libusb-win32 device driver
and it works fine.

> The ones under Universal Serial Bus Controller come and go as the Olimex USB
> is plugged and un-plugged and are Microsoft Drivers. I don't know what these
> do.
>
> So the one with the yellow question marks are probably the problem. If I
> delete it or disabled it it comes back on the next h/w scan. When I try to
> install the driver using the *.inf that Freddie supplied, I see a windows
> error dialog stating something like "driver does not describe your
> hardware".

You have to install the original FTDI driver for that (serial port).
After that, the wizard will not pop up again.

> We are not using the serial port built into the Olimex.
But you still need to install the FTDI driver for it. If not
the Windows Wizard will come out. You can use Device
Manager, right click the device with the Yellow Question
Mark and then use Update Driver to point it to the
FTDI driver (provided by Olimex).
