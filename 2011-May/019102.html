Hi,
I've been working on support for the freescale dsp568013 and 568037.

 0001-partial-support-for-568013-and-568037-target-integra.patch
The first patch supports read, write, halt, resume, and load_image.

0002-flash-support-only-full-erase-write-for-568013-and-5.patch
The second patch includes flashing support, write_image, erase_sector,
add/remove protection, erase_check.

I have a couple of things on which i'd really appreciate getting some
feedback and/or suggestions:
- flashing speed: currently i flush the jtag queue all the time, to
make coding easy. flashing 4KB currently takes about 45secs, which is
not acceptable. the algorithm i'm running requires openocd to check a
register (written by the core) to see if it can send more data. if i
queue commands on jtag, how can i keep this working? won't all the
commands get shifted into jtag and eventually clog the system?
- memory addressing: each memory address corresponds to a word. i did
not find a way to get openocd to handle this, so take the addresses
given by openocd and divide them by 2, and put 2 bytes at each
address. any suggestions on how to make an elegant version of this?

thanks!

--
Rodrigo.
