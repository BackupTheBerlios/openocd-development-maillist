Jon Povey wrote:
> Especially in the case of OpenOCD I can see no reason for changing
> to use units of 1000 bytes. We are computer programmers. We want
> powers of 2.

For me it depends on the use; if measuring storage then yes I expect
powers of 2, but if measuring transmission speed then I prefer powers
of 10. Think Ethernet, USB or I2C.

I think low-level transmission usually stick with Si-prefixes.

Maybe it's even simpler - if counting bits then it's base 10, if bytes
then base 2.

Just some reflection. I don't feel strongly. :)
