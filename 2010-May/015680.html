It actually does not matter to me. But I feel strange to use kibibytes,
which is used by very few people.
http://en.wikipedia.org/wiki/Kibibyte

to a power of ten as well as a power of two".

For example, I think Windows in general use KB to refer to
KiroBytes. I just right click one file under Windows XP, it states
that the file size is 273 KB (280,269 bytes). So I will think
it is quite okay to use the existing method, no need
to change. If we really want to change, then probably can use
the Windows method (to add bytes in bracket).
