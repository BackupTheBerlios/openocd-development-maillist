Hello Vincent,

I've just checked in a change to the image code that supports multiple 
sections within an image. I tested the code using an ihex file containing a 
gap, thus forming two sections.

Image writing has been removed alltogether, because this is hardly useful, and 
writing anything more complex than a ihex file would probably be too complex 
for our purposes.

I added a "flags" field to the sections structure which is currently unused, 
but could hold additional information from ELF sections like readable, 
writeable and executable, should this be necessary.

Regards,
