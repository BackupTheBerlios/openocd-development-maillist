Hi,

The attached patch (thumb_disassembler.patch) implements the disassembler for 
thumb mode (according to ARMv5 instruction set).
I've checked the correctness of the text output for most opcodes against 
objdump disassembly.
A remaining issue is the bl/blx instructions which use dual opcodes, as no 
context is stored, they are currently displayed as a "prefix" instruction and 
a bl/blx instruction.

I also join another patch ( disassembler_beautify.patch ) which gives a more 
readable (very subjective topic) disassembly output :
- it displays "mov r0,r0" as NOP
- it removes null offsets.
I also add a comment giving the "shift" field values.

Vincent
