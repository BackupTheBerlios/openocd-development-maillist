On 02/04/2011 18.32, Øyvind Harboe wrote:
> How did you test?
> Do you have a list of features that work and doesn't work for pandaboard now?

Hi Øyvind,
I just did some basic testing:
- halting processor
- read/write memory/registers
- stepping
- resetting CPU
- setting hardware breakpoints
- resuming processor
- read memory through APB

I did this just to check that there is no regression respect to previous 
versions.

Features still to be implemented on PandaBoard (and all others 
Cortex-A*) are:
- software breakpoints
- tracing
- a reliable way to use two or more cores on cortex-A9
- maybe many others ;-)

NOTE: I want to point out that I'm working a lot with bootloaders so I 
didn't test it very well with a real OS running or with GDB.

regards
Luca
