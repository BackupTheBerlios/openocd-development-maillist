hi,

i've attached a patch with partial support for the dsp5680xx.
halt, resume, mdh, mdw, mwh, mww, load_image, verify_image, should work.
there is an issue with the addresses displayed. openocd uses byte
addressing, but this uses word addressing, so every time a new line of data
is displayed (mdh 0xXXXX #, with #>16) then the first line will correspond
to the data at 0xXXXX, and the second line will say 0xXXXX+0x20, when it
should be +0x10. haven't worked this out yet, i'll check the mails regarding
this issue in the dsp563xx.

the support is partial, i'm working on it.
i wanted to see if i could get some feedback.

thanks!

Rodrigo.
