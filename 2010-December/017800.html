Every NAND driver keeps a private copy of "struct target".
This code arrangement does not permit further optimization
since each driver have to refer to its own specific
private data.
The set of patches below, moves "struct target" in the
common NAND driver data, and forces all driver to use it.

I do not have the right HW to test the patches on each target.
I decided to split the patch per target, to make code
review easier.
