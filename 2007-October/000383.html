Hi,

Find attached a patch for review.

Firstly it adds a new command - reset_type [reset_hard|reset_soft]
Simply enable user to select whether we use default behaviour of hardware
srst or a software reset.
This is a global target option, so effects all connected targets.

Also included are some fixes to the cortexm3 reset behaviour.
The current behaviour had a broken reset run, it would always halt at the
reset vector.
I have tested on stm32 but have no luminary targets to test - could either
Magnus or other luminary users please check it does not cause problems.
