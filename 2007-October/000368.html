Find attached a verify patch for review.
Simply adds a new command verify_image that performs a crc32 on a memory
area and compares against a image file crc32.

If it can allocate a working area then it will use a loader to calculate the
crc otherwise it will perform a direct memory read - the loader is much
faster on larger files.
