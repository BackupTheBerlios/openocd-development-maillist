Hello,

the first support for GDB flash writing was based on my job,
I was thinking about implementing qXfer:memory-map:read but I have
not started any work on this so far.

Just a few ideas I had when I was thinking about the implementation:
First implementation may be just very simple, sending data from a file edited by hand.
Another approach is to generate the memory map from a template and from
definition of flash banks.
However there is currently no similar structure describing RAM regions as
for flash banks, but this should hard to add.

The good point is, that OpenOCD does not need an XML parser, as it does not need to understand the XML
description of the memmory map, just to generate it.

Regards, Pavel
