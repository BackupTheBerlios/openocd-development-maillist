Hi,

Attached is a proposed patch to add a gdb_detach config option, giving the
user a choice of what they want openocd todo when it receives a gdb detach
packet.

Default behaviour will be as it is now - target is resumed. But you also
have the option of the following:
gdb_detach resume
gdb_detach reset
gdb_detach halt
gdb_detach nothing - leave target as it is (couldn't think of a better name)

Also included in the patch is a cleanup of the arm966e cp15 command.

Cheers
Spen
