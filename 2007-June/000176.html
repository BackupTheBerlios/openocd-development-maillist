On Thursday 31 May 2007 23:19, Spen wrote:
> We may have to include our own version of elf.h - it is not included with
> mingw and some of the defines are missing for cygwin (ELFDATANUM) being one
> I have found.

As we use only a few structures of elf header, we might include the necessary 
definitions in replacements.h.
A patch (elf_mingw32.patch) with the proposed changes is attached.

I have tested it with :
 - linux (native)
 - linux (cross compilation)
 - mingw32 (cross compilation)
 - cygwin (native)

I also need to modify a few bits to be able to cross compile for mingw32 
target (see fixes_mingw32.patch) :
 - "far" is a reserved keyword for windows compiler and cannot be used as a 
variable name
 - modify include order to avoid redefinition of ERROR() macro
