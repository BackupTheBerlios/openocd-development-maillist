The schematic in my thesis is buggy. It has nTRST inverted while genuine 
Wigglers and correct clones simply route nTRST through a buffer. Using the
"wiggler_ntrst_inverted" cable definition works around this bug.
>
> After some struggling, I got the combination of Olimex USB + IXP425 running
> (which was my main goal) - I can talk to the target, display memory, etc.
>
> When I try the same using the parport interface, initialization of OpenOCD
> fails quite early:
>[snip]
>
> Hooking a scope to the parallel port pins, I see that TRST and SRST are
> never activated during startup. I know nearly nothing about JTAG, but I
> would have expected a reset pulse first before any other checks get run.
> Is this correct, or am I missing something?
>
The OpenOCD normally doesn't assert nTRST, which is an optional signal anyway. 
The JTAG port gets reset by applying 5 TCK cycles with TMS held high.

> As a side note, some extra sample config files (or updated documentation)
> might be really helpful for first-time users - I had to pull the target and
> jtag_device parameters from the source code.
>
There are example config files in the doc/configs/ directory, and a growing 
number of tested configurations are available at 
http://openfacts.berlios.de/index-en.phtml?title=OpenOCD_scripts. The Wiki 
also documents most of the configuration and user commands available.
