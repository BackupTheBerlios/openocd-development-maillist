The   target->breakpoint_add function does not really add a breakpoint,
it only checks  if it is possible to add a breakpoint.
For targets where  it is possible to add and remove breakpoints while
running  it is good to be
able to actually setup the breakpoint at this time, at least for
hardware breakpoints.

So my suggestion is to allocate and prepare the breakpoint_t structure
and pass it to  target -> breakpoint_add.
If the call returns without error then it is added to the  breakpoint
list or be deallocated if the call fails.

This gives the target the possibility to change the type between hard
and soft and also to setup the breakpoint immediatley.
