I'll send these out for comment, and merge tomorrow if nobody objects.

 - Make the "examined" flag be per-target.  This is a bugfix, but
   the bug wouldn't have been seen without multiple CPU targets.

 - Two arm7/arm9 memory utils aren't actually arm7/arm9-specific;
   they're generic for anything that handles ARM algorithms.
   (Example, the Cortex-A8 uses them.)

 - Cleanup the ARM11 method table.  Includes removing another
   annoying macro, and making more code file-private.

 - Make ARM11 use those "new" memory utils.  This accelerates
   things like "flash erase_check" and GDB memory checksumming.
