I still don't see a problem.  Few scripts are portable between
CPUs; and fewer still, without actually having been used and
tested on multiple CPUs...

Your scenario is "script got the 'phys' token wrong, and then
it broke when used with an active MMU".  Right?

It's very hard to write "generic" scripts that behave with an
MMU active, and we can't do anything about that.  One basic
issue is that the memory map needs to be set up to match what
the script expects, but OpenOCD has no control over such maps.
