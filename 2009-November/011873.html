On Sunday 01 November 2009, David Kaplan wrote:
> I'm trying to get u-boot running on a smdk2410-based board.
> I'm using openocd and jtag and am trying to load u-boot into ram as follows:
> 
> load_image u-boot.bin 0x33f80000
> 
> Which *seems* to work:
> 103668 bytes written at address 0x33f80000
> downloaded 103668 byte in 0.500045s
> 
> I then try to run it by using resume 0x33f80000 but it doesn't seem that
> u-boot is executing.
> What am I doing wrong?

See the "Project-Specific Utilities"
section near the end of

 http://openocd.berlios.de/doc/html/OpenOCD-Project-Setup.html

Looks to me like you're missing the "reset-init"
event handler for your target, which will need
to set up the clocks and DRAM controller.
