>     But here comes another question, it seems that AUTO PROBING could 
> only detect one TAP.

As in, that's all there is.


>     Such as beagleboard, it provide not only DAP TAP but also JRC TAP.
>     How could I make sure it only has one TAP?

I don't understand your question.  Those TI chips only
have one tap ... until you ask the JRC to enable others.

So what you do is check the docs for that 0x1ba00477 TAP
to see what it does.  In this case, I see you have some
old Cortex-M3 revision (0x3ba00477 is r1p1) ... I thought
you said you had a Cortex-A* part?  M3 is *very* different.

So far as I know, nobody using Cortex-M3 has so far felt
the need to use a JRC.


>     How do I know this TAP is for DAP other than BS?

You will at some point need vendor documentation.
