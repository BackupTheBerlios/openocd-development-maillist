On Monday 02 November 2009, Pieter Conradie wrote:
> ...
> SVN2645: works
> SVN2646: broken
> ...
> It appears that SVN2646 did break it.

So this *is* a regression.  That SVN id matches
efef05870d726fe4cb6786d785fae4628fe7ec1e, also
known to "git describe" as v0.2.0-125-gefef058;
so it happened 125 commits into the 450+ which
preceded the RC0 tag.

This seems like a "must fix" for 0.3.0 ...

In the longer term, it would be nice to switch
more of the downloaded ARM algorithm code over
to the scheme where they terminate with BKPT
instructions not loop-to-self.

Then anything on an ARMv5 (or newer) core -- like
sam9260 and lots of other chips! -- won't need to
use the code which is breaking here, and will run
a bit faster because it won't need to get/set a
breakpoint.  The ARMv4 stuff will continue to
need hardware breakpoints, of course.
