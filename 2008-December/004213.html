> Thoughts?

I think it would be helpful to define some long/mid-term goals and
meanwhile delete bits of code that are old/complicated/unecessary
that is making any other changes harder than necessary(ref my post
about deleting the in_handler, in_check stuff).

Note that there are some capabilities in the current JTAG API
w.r.t. e.g. hardware FIFOs that I'd hate to see thrown out with the
bathwater. Also, it performs *very* well with feeble CPUs like an ARM7 ;-)
See the MINIDRIVER stuff for how feeble embedded CPUs can
make OpenOCD fly.
