Having looked at this for the beagleboard, converting a BDI config  
file to our target infrastructure isn't all that easy.  I have a few  
ideas of how to rearrange the ARM targets so the actual target is a  
much smaller amount of code that is similar to what the BDI config  
files are doing.  I'm still reading up on the ADIv5, but I think we  
can restructure the target code in terms of the address maps that are  
exposed (APB, AHB) and have all of the ADIv5 targets share a common  
code base otherwise.

I want to investigate this _after_ 1.0.
