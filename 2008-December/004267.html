I've been working with the SAM9-L9260 board with the Olimex ARM-USB-TINY
JTAG device.  I'm working in Ubuntu 8.10 with libftdi.  Everything has been
working fine, except for 'reset init' and 'reset halt'.  All memory commands
like mww and mdw have been working, but when I try to do a reset with a
halt, something happens that messes up these memory commands.  Various other
commands, like 'targets' or 'poll', work fine before and after the reset
halts.  I have tried doing 'reset run' followed by 'halt' and all commands
still function.  I have also tried running the reset-init event as a
seperate script and it works fine.  I've attached the configuration files
used or included as well as the debug level 3 output from OpenOCD when the
'reset init' command is invoked.  Thanks for any help.
