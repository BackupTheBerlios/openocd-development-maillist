This patch adds JTAG state tracking to dummy.c


It deliberately use logic different from the lookup table that any 
calling code would use, so as to verify the state transitions with 
another point of view.


I also introduce the typedef for enum tap_state as tap_state_t which is 
one word shorter.

The means of the logging and printing of the state transitions is 
debatable.  I don't care, this can be easily commented out by someone 
who is annoyed by it.
