We have very little documentation on the JRC (Icepick).  We have  
discovered the IR value for the ID code (0x4) so we can re-examine the  
full chain successfully.  That also means that OpenOCD will shift out  
the correct number of bits when it thinks the DAP is turned on.   
Beyond this, the documentation just has a pseudo-code description of  
how to enable the DAP with no commentary on what each step does.

We have a DAP tap-enable event started that runs successfully but does  
not appear to actually enable the DAP.  After telling OpenOCD to  
enable the DAP, our experiments have shown that only the JRC is in the  
chain.  The next step is adding a 'runtest 10' to the end of the event  
to match the sequence laid out in the documentation from TI.
