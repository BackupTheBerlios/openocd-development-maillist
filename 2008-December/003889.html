OpenOCD SVN has a slight problem with the layout now that we are
going to  work with release branches.

The problem is that trunk contains OpenOCD without any subdirectories.

Next to trunk there is a zy1000 project.

The zy1000 project should be branched in the same manner as trunk.

The raison d'etre for the zy1000 is that it contains a lot of *BIG* files
that are used only in embedded development.

Possible courses to take:

a) do nothing, but remember to branch both trunk *and* zy1000 directory
b) push trunk into trunk/openocd and zy1000 into trunk/zy1000.


Comments?
