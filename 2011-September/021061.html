Hello,
we have a set of patch for providing linux ARM awareness support.
This linux awareness relies on some function implemented by for RTOS.
The RTOS interface was too generic for the linux awareness, then we have done some changes on RTOS :
- overload gdb_thread_packet functions from RTOS,
- introduce smp support,
- rtos ps command.
This feature provides the display of kernel stack from all linux threads (usr or kernel)
- This is done correctly when system is stopped in kernel context.

Best regards
