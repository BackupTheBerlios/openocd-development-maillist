I now that this solution exists, but lets say I'm lazy :).
Using VDMLPT will require much more changes in openocd and lot of tests.
Openocd on Windows uses port address to recognize which port to use e.g 0x378, 0xe800.
VDMLPT I think need to be recognized by names VDMLPT1, VDMLPT2, VDMLPT3 so:
- all existing configuration files should be changed to replace 0x378 with VDMLPT1
- configuration file parsing mechanism must handle those new names
- I think VDMLPT device aren't supported in win9x so that way or another 0x378 kind of numbering still is needed.
- changes in documentation
I'm also not sure if PCI card with LTP port also create VDMLPT device.

Probably those naming issues could be somehow workaround-ed and still using port address number, but this is not a quick task.

It was much easy for me to use inpout32 that has very similar API to giveio, and I could write simple patch in few minutes.
