Perfect. git is distributed, so when you create a commit it is stored
locally in your repository, and then format-patch can be used to
easily create patches from all commits that you have locally but
which were not included in what has been downloaded from the upstream
repository.

The command git send-email is also handy, it can be used to send
patches out directly from git, after some configuration.

Please remember to create one commit for each logical change. And
configure your name and email:

git config --global user.name 'Your Name'
git config --global user.email your at email.here

This name and email is stored as the author information when you
create commits, and it is included in patches, so that the correct
author information is available also after your patches are included
in the upstream repository.
