The stm32w, em351 and em357 uses the same set of register for
controlling the FPEC as stm32x, but these registers are located at
different addresses. There are also some differences to the option
bytes that make me think that the code would only be worse off if I
tried to support these devices in the stm32x driver. I also don't know
of a way to identify the chip type yet, reading the bytes used in
stm32x_probe to determine the chip only returns zeroes on my em357.
