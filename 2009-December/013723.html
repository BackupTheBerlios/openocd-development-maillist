Contents are cleanup, a bugfix, and a new feature that I hope
will enable usage that can finally get this code to work:

 - Move an ETB-specific parameter to the ETB code.

 - Clean up some ETM_CTRL register handling, and use the
   correct bit for full/half speed clocking.

 - More cleanup, getting rid of a partial ETM_CTRL copy

 - Add "etm trigger_debug" command, effectively letting
   the ETM trigger breakpoints, watchpoints, or more
   complex debug entry events ... without needing a trace
   port driver to work.

These pass sanity testing.  Since as far as I can tell
nobody is able to use the ETM for Real Work (tm) yet,
I'll just merge these ... they can't make things worse.
