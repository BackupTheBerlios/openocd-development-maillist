I've looked over this patch and I'm inclined to commit it.

W.r.t. formatting there are two nits:

- you seem to be using tabs = 4 spaces, whereas the committed code
uses tab's.
- You are using Tcl correctly in adding a  ; before the # comments
at the end of the line. OpenOCD has a special "hack" to support # as the
comment char at the end of the line. See example from tclsh below:

% puts abc
abc
% puts abc # asdlfj
bad argument "asdlfj": should be "nonewline"
% puts abc ; # asdlj
abc
