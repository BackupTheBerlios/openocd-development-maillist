On Saturday 11 August 2007 15:15:19 Pavel Chromy wrote:
> Hello Dominic and everyone on the list,
>
> there are some serious bugs in 192 causing OpenOCD to crash (memory access
> violation)
>
> target_request.c: function delete_debug_msg_receiver() dereferenced
> uninitialized pointers p and c, causing a crash, fixed by this patch.
>
> image.c: image_add_section() assumed that there is already at least one
> section in the image and tried checked whether to append data to it, which
> caused crash by dereferencing invalid pointer in case there was no section
> yet.
> the patch fixes this and makes the code a bit more readable by adding a
> local pointer instead of accessing the same array member several times
> using an index decremented by one.
>
> Best regards,
>    Pavel
