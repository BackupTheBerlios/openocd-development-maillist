Hi,

here's a patch which allows openocd to build on the Debian GNU/kFreeBSD
platform (i.e., GNU userland, Debian package management, and FreeBSD kernel).
See http://www.us.debian.org/ports/kfreebsd-gnu/ for details.

I'm currently disabling --enable-gw16012 and --enable-amtjtagaccel on
Debian GNU/kFreeBSD, as these targets don't support FreeBSD at the
moment (but I guess they could be fixed similar to the other files).

As for the patch: Debian GNU/kFreeBSD uses __FreeBSD_kernel__ as
hardcoded macro which we need to check, plain FreeBSD uses __FreeBSD__,
so we need to check for both.


HTH, Uwe.
