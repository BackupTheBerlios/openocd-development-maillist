Magnus Lundin wrote: 
> Trying to write to RAM from the flash writer code should (IMHO) give an
> exception and an error message that the target address is outside flash.
> It usually receives  errors from the flash controller.
>
> The flash loader should be able to know if a section is BSS and/or 
> supposed to be in
> RAM and disregard those sections.
Agreed - I disregarded the difference between uplading to memory and 
flash programming - when the code can distinguish these two, the flash 
code can discard all sections outside flash, while they should be 
preserved and cleared when writing to RAM.

> Still we must think about how to handle absolute addresses as in elf 
> and hex files or
> relative addresses as in bin files loaded to a specified offset (flash 
> start address).
Hm. When writing binary files to flash, I think it is quite useful if 
the file is written to the beginning of the flash if no offset is 
specified. On ELF files, I would expect the file to be written to the 
addresses as specified in the ELF file when no offset is given, which 
might be considered a different behaviour. Special considerations might 
be needed when one ELF file contains sections that map to two ore more 
different flash devices/areas.
