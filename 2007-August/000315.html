I havnt found any similar register for the Cortex-M3, but I dont think it
is needed.

The JTAG debug port can read and write any memory mapped lo9cation, and
even CPU registers using Core Debug registers, while the processor is
running so it is very simple to create a "DCC channel" by using a word or
block in RAM for data and another word in RAM for flags and
synchronistation. So we can effectively use Cortex-M3 RAM as shared memory
betwen OpenOCD and the application running on the target. Normal IPC
design for shared memory blocks.

I have made some experiments downloading data to an application running on
Cortex-M3 using this principle and it works. There are some issues with
the  timing of Debug Port and AHB-AP since memory access can take longer
when a running application is using the bus systems on the Cortex-M3
compared to when the processor is halted.
