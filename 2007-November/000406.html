If CPSR contains garbage in M0-4, then OpenOCD will simply exit.

However, a reset of the CPU is sufficient to fix this, so it seems wrong to exit
OpenOCD.

I'm still trying to figure *why* CPSR contains garbage. E.g. 0x100e984, which is
a valid ROM address on the target, at a guess I'd say that CPSR contains the PC,
which kinda makes sense because PC was the last register to be read before CPSR.
Seems to happen after many "reset halt" in rapid sequence. I guess
I'll dive into arm7_9->read_xpsr() and see what I find...

Comments?
