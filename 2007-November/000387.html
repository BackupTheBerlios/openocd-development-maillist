On 11/5/07, Spen <spen at spen-soft.co.uk> wrote:
> >
> > The OpenOCD commands can potentially take a log time, e.g.
> > programming flash or executing load_image.
> >
>
> Using the monitor interface is not really the correct way to program flash
> in my opinion.
> When the support for qXfer:memory-map etc is added, then this will not be a
> problem.

Flash programming is a bit of a non-sequitor here. The core of the
problem is that OpenOCD supports long running monitor commands only
after a bit of tinkering.

I don't think there is a "right" and a "wrong" way to program flash.
There are two different methods and both have merits. OpenOCD should
support both without producing error messages and leave it to the
users of OpenOCD to choose which method that they think is best for
them.
