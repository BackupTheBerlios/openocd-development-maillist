Hi,

To find (stupid) bugs in my own modifications of openOCD code, I would like to 
rely on compiler warnings.
Current build files do not define CFLAGS at all, so autoconf puts default ones  
( AFAIK "-g -O2" on all hosts) if nothing given on the command line.
So I think it could be nice to add "-Wall". I attach "Wall.patch" but I'm not 
an autoconf expert at all, so the nice way to do it  might be slightly 
different.

For those interested, I also fix all current warnings (some of them are just 
gcc nonsense ) to get a better SNR to detect my own bugs.
See fix_warnings.patch attached.

We can also detect minor incorrectness on variable arguments in log_printf() 
command by giving a hint to gcc that we expect some printf-like arguments.
( see check_format.patch attached )
I fix these minor bugs in "fix_format.patch".
It remains several noisy "zero-length printf format string" warning (which are 
of course not bugs but gcc ignorance), it might be useful to prevent them 
(see avoid_null_string_warning.patch)

-- 
Vincent
