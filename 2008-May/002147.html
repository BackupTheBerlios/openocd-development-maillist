Hello Raphael,

Raphael Bustin wrote:
>> The main reason being that openocd supports more than just the ftdi devices.
>> It uses a common jtag layer.
> 
> Interesting.  What other (USB<->JTAG) chips are supported?
> AFAIK, the FTDI chips are ubiquitous in this application -- 
> they're in several of the commercial "wigglers", etc.

You are right, they are. However, it is not very wise to limit OpenOCD's functionality to a single
USB chip, moreover using a closed library with no free (as freedom) alternative with equal functionality
(usage of proprietary libraries was discussed on this list before).
I believe, that FTCJTAG covers only interfaces which are built on bare FT2232.

To your question, PRESTO is first (and hopefully not last) supported USB<->JTAG interface which can *not* be used
with FTCJTAG, despite there is an FTDI chip inside - it's an FT245 and it is used just as USB parallel FIFO,
with custom logic hooked behind it.
