I can't seem to get ftdi_usb_open_desc() to work on freebsd, ever,
even with supposedly matching strings.  I've included a patch that
will try ftdi_usb_open_desc(), then fall back to ftdi_usb_open()
[which always succeeds on fbsd] if it fails.  Hopefully everyone will
find this acceptable enough to commit to svn.  I think it makes the
system more robust for beginners, even on systems where
ftdi_usb_open_desc() works as intended.  I've given the new code the
'bogus' revision 659 in the patch file - I'm no code guru; seemed
reasonable.

Thanks,
