I'm thinking that it makes sense to add GDB attach event scripts
to the target library.

Minimally this achieves two things:

- removes a string of "monitor" commands in GDB connect scripts,
convenient
- enables users to profit from tested procedures rather than having
to try to put this together themselves.

LPC2148 being a good example that these scripts have to
be absolutely correct to get anywhere and have a stable
system.

I'm also tempted to disallow configuration of the interface in
the same script as target definitions. This would encourage
building the target library. Of course users should be allowed
to do anything they want, but contributing back good
target scripts would help the community.

Thoughts?
