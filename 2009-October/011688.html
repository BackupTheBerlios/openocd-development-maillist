On Monday 26 October 2009, Øyvind Harboe wrote:
> Anything else we need done to the code before 0.3?

It's "bugfixes only", so far as I'm concerned.

I put that in quotes because for now we're not as
strict about that as we should be next week.  New
features should not be merged.  Cleanup ... minor
stuff is OK, if "it really can't break anything".
And doc updates, of course.

As we get a bit closer ... we should look harder
at bugfixes, to see if they shouldn't wait.  When
we see regressions, they should get fixed ASAP.


Everyone should be testing all the hardware they
have handy ... at least sanity testing it, if not
more aggressive testing.
