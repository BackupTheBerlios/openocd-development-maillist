On Tuesday 20 October 2009, David Brownell wrote:

>  - From time T+2 and later, everyone will want to get new clones
>    of that repository.  I'll send out an email announcing this,
>    presumably as a followup on this thread.

That's where we are now ... get new clones, from the main
repository.  (I've not done anything to the mirror...)

That is:  scrub your old copy of the GIT tree, unless you
have local commits saved and you want to transfer them.

Then "git clone ...".  That'll only be about 16 MBytes.

If you _don't_ create a new clone, you'll get a merge commit
which glues the old and new branches together ... so your
local tree will forever be a hundred or so MBytes bigger than
it needs to be.

- Dave

     I'd gone back to the original plan.  This was an issue the
