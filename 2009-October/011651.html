I was just poking around the repository this morning looking for things
related to the Samsung microcontrollers and I noticed the old naming
conventions have some how disappeared from the TCL configuration scripts.
It also appears that redundant interface configuration files have popped up,
specifically for the Olimex JTAG Tiny interface (one with "olimex_" and the
other without).  At the same time, I noticed that some of the interface
configuration scripts didn't have their manufacturer's name in front of
them.  I thought the standard was something like

manufacturer_jtag_interface

using only underscores.  It appears that hyphens have appeared.  This
problem has also appeared in the board configuration scripts.  It might be
good to get everything standardized before 0.3 release.  I can take some
time to go through it later if no one else has the opportunity to do it.  It
may be wise to hammer down a standard and document it some where in the
source tree or in a developer document.
