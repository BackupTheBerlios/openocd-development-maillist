I just did a clone of the git repository and noticed how long it took to clone one of the many objects. After a little sleuthing, I've determined that:
2869bc4... Adds a few small files
2f37d16... Adds several large files.
af3b53a... Adds more large files. It immediately succeeds the previous
1ef7ccc... Removes all files added by previous three commits and nothing else

Removing those four commits brings the repo from 99MB to 5MB. While doing so would destroy history that others' use, it would be a great boon to those who are cloning it for the first time, and would not be difficult for the rest of us to synchronize with.

Any takers?
