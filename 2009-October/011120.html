This file allows programming of this chip. In fact it can be done via
xsvf without even defining a tap, but only if it is the only device in
the chain. This tap config is needed when it is in a chain.

Please stick this file in targets.

(I get the impression very few people are using openocd for cpld
programming because there is much breakage - more about that in later
mails)

-------xilinx-xcr3256.cfg-----
#xilinx coolrunner xcr3256
#simple device - just configure a tap
jtag newtap xcr tap -irlen 5 -ircapture 0x01 -irmask 0x1f -expected-id  0x0494c093
