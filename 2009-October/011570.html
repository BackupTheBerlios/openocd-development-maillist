Øyvind Harboe wrote:
> Should commands that *could* be supported by targets be added
> even if the OpenOCD target code does not support it them?
> 
> If users are searching for the right way to do mrc/mcr operations
> for, say, XScale, would it not be better if the command was there
> but returned "not implemented"?

Not really. That clutters the output of "help" and makes it more 
difficult to see what commands *are* supported.

Having a command exist but return "not implemented" won't help the user 
- someone needs to implement is anyway, and a developer who is capable 
of implementing such a command will not need the hint.
