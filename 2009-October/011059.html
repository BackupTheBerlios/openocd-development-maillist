Hi all,

The Subversion repository should now be treated as read-only.  

As David's recent post indicated, there are some minor administrative
glitches that prevent us from enforcing this intended policy, but this
should be fixed at some point.  In the meantime, I hope that the last
revision (or these announcements) will be sufficient to prevent any
developers from committing further revisions to Subversion.

All new development will be pushed into the GIT repository hosted by
SourceForge.net [1] (and mirrored by repo.or.cz [2,3]).  Happy hacking!
