Jon,
Gnu ld has the option -nostartfiles to stop it from including startup files so you can have your own startup sequence. Next you'll need a really small C library. Forget about uclibc. I've compiled the small C library from MSPGCC (for TI's msp430) for ARM. This should get you executables with a reasonable size.
