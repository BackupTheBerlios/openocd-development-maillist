You are correct about that one.  I was just curious.  Plus, I was
considering doing some revisions on the configuration files to centralize
some of the items that seem to be redundant.  As an example, look at all of
the boards that configure the clock for an AT91SAM9 of some form.  The
configuration is practically the same.  The only difference may be the
crystal frequency.  I was thinking it might be helpful to centralize some of
that type of logic to a TCL function that resides in a target file instead
of repeating the logic for each board that uses the target.  This may show
up in the future as I play more with NAND on my SAM9-L9260 board.
