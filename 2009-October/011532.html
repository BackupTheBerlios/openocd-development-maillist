Hi all,

The following series of patches present new GIT-based release scripts.

The associated documentation that describes the process requires work,
but some parts have been updated.  Please provide feedback after testing
them on your own clones.

For testing, the last patch provides a test script that will clone your
repository and runs release.sh to build a hypothetical release history.
This test script should evolve into a full unit test before I push it.

In short, the script should be run with the release action requested and
the type of release cycle that will come next:

  tools/release.sh {release|branch} --next {major|minor|micro|rc}

Context of various usage can be seen in the test script, including
demonstrations of the options used to manage RC releases/branches.
I'm not entirely satisfied with it, but it seems to work okay for me.

Cheers,

Zach
