I would like to see reset_config's in the target config
scripts so that the target config scripts are useful
standalone against PCBs that use the part in unclever
ways.

The only nasty side effect of this is that the board
config script has to be kept in sync with changes
in the target config script.

My solution to this is quite simply that the board script
should completely override any reset_config settings
in the target script. Break the "communication" between
PCB and target scripts here, it's more documentation
to read and more things to go wrong for marginal
benefit, IMHO.
