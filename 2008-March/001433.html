The target library should contain configuration which is
in regards to the target only.

reset_config can be used to set up reset configuration for the
interface as well as the target.

In light of the recent discussions on reset_config and whether
to change it to be a better match for the target libraries requirement,
I believe that the best way forward would be:

- document reset_config better. Especially the documentation
  should contain more about the awful truth.
- each reset_config statement in the target library deserves
  a comment, it is a sufficiently complex issue.
- reset_config is used in the target library to describe only the
  targets reset requirement. However when trst is not required
  do  not use trst as some interfaces do not have trst hooked
  up.
- if the user has an interface or cabling that conflicts with the
  reset_config statement in the target library, then he can
  override the target library using the -c or -f options.
- perhaps the reset_config command could be simplified,
  but it is necessarily complex. Leave reworking
  reset_config be until ideas have matured and moment
  of clarity comes.
