I guess we're all well aware of the advantages of a clean C++ design.

The biggest argument from my side against C++ is that the g++ compiler is just 
soooo slow. A fresh compile on my 1.8GHz machine already takes nearly 30 
seconds - no doubt this will rise to minutes if we used g++ instead of gcc.

But it's also not as if writing _good_ C++ code (the one that works on 
embedded systems as well) is any easier than writing C code. Personally I 
find it a lot easier to get C code right, which is why I chose this language.

According to sloccount OpenOCD reached 48,553 SLOCs - I doubt a project of 
that size could easily be rewritten to using C++. And mixing C and C++ is 
like calling for the worst of both worlds - the legacy code would still 
suffer from missing compile-time checks, while all of the code would suffer 
from g++'s awfully slow operation.
