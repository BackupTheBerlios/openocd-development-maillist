Hello,

Øyvind Harboe wrote:
> I'd like to retire JTAG_END_STATE now that end state
> is tracked by the jtag_xxxx() layer, i.e. this code no longer
> has any effect and is just complicating things.

I agree with this. The only JTAG queue commands which make use of end_state are
JTAG_SCAN, JTAG_STATEMOVE and JTAG_RUNTEST, but they do so only if no explicit end state is given
(which actually never happens - guaranteed by jtag_add_xxx() layer AFAICS).

Thus I belive this change may be done, enabling further simplifications to take place,
i.e. remove JTAG_END_STATE from the drivers and then eliminate end_state variable completely.

Regars,
  Pavel
