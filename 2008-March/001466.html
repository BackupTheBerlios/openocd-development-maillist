On Thu, Mar 6, 2008 at 9:47 PM, Dominic Rath <Dominic.Rath at gmx.de> wrote:
> > > - code should not produce any warnings with:
>  > >
>  > > CFLAGS="-Wall -O3" configure
>  >
>  > Fine with me, must admit i always use O2 level - mainly as this is the most
>  > tested gcc level.
>
>  -O3 is calling for problems. It is known to be broken for several gcc
>  versions, and usually offers little improvement over -O2.

The reason I suggested -O3 to find warnings is that it will enable
more analysis of codepaths and thus come up with helpful and
not so helpful warnings.

I'm not suggesting OpenOCD should be compiled with -O3, though I
can report that it works fine w/gcc 3.4.x arm.

>  I'm still in favor of leaving the CFLAGS completely untouched - -Wall -O3
>  doesn't help a developer, because you can hardly debug -O3 code (other than
>  running test cases, once everything works at -O[0-2]), and anyone else isn't
>  interested in the warnings.

The current thinking is to have an item in the coding standard what options
that code should be warning free with.
