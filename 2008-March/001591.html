> >  >
> >  > Added jtag_khz. It's the same as jtag_speed, except that 
> it takes  
> > > KHz as arguments. JTAG interfaces can implement a jtag->khz()  > 
> > translation function that translates KHz to speed value. By 
> default  > 
> > jtag_khz will show an error message if jtag->khz() is not 
> implemented.
> >
> >  You forgot to attach the patch :-). I'm curious as to how inexact 
> > mappings  are to be dealt with. e.g. for a jtag_speed which is only 
> > expressed as a  divider of a 6Mhz clock, what happens if you have 
> > jtag_khz 4500. Round down  or error?
> 
> It's maxium speed in KHz. 0=RTCK. The commands prints an 
> error a failure if it can't support the kHz/RTCK.
> 
> There are several reasons why an interface might fail:
> 
> - no way to know the maximum KHz
> - interface has a minimum speed
> - no RTCK support
> 
> 

we need to keep the docs (openocd.texi) upto date when  new commands are
added.
