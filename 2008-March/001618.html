The problem here is that the jtag_add_reset() has an
awkward API.

The precise error number is used to interrogate the
target/interface as to the method that is to be used for
resetting the target.

My preferred solution would be to make jtag_add_reset()
return void like the rest of the jtag_add_xxx() functions
and require the caller to check the reset type before
invoking jtag_add_reset().

This is straightforward to do and is done in other
places, but I didn't want to change the API at the
time, because it would require retesting and the
timing wasn't right.
