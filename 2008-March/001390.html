My understanding is that %zu and %zx are specifically included in C99 to 
deal with size_t (there is a similar flag for ptrdiff_t).

GCC produces these warnings when you use size_t but not when you use the 
underlying int type.

inttypes.h does not produce the %zu syntax because it only deals with 
the various generic int types.
