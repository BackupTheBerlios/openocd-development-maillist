On Sun, Mar 2, 2008 at 8:46 AM, Michael Bruck <mbruck at digenius.de> wrote:
> What is the point of this? I use // comments to signify experimental
>  changes in progress vs. /**/ that signify comments. Your patch removes
>  that important distinction.

OpenOCD is an open source project, so we need to stick with the
consensus. For now that consensus is to use ANSI C. Anyone is
free to try to wip up enthusiasm for a different approach, though there
is evidence that moving away from C is an uphill struggle. Changing
the mind of Uwe will not make a difference, you have to change
the consensus. I don't even recall if Uwe has an ANSI C preference
as such :-)

Uwe provides a lot of necessary details e.g. w.r.t. licenses and documentation
and it is natural that he would then also point out discrepancies
in OpenOCD code relative to the coding style consensus. His input
is highly appreciated! :-)
