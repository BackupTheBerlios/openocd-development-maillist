from jtag_add_plain_dr_scan():

	if (cmd_queue_cur_state == TAP_TLR && cmd_queue_end_state != TAP_TLR)
		jtag_call_event_callbacks(JTAG_TRST_RELEASED);

this code is broken as a scan will go via other states that release TRST.

However, no code uses JTAG_TRST_RELEASED, so my instinct is to
remove JTAG_TRST_RELEASED as fixing it is non-trivial and the code
already written is pretty far from a working solution anyway.


Any objections to retiring JTAG_TRST_RELEASED?
