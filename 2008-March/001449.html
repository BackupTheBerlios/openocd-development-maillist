On Thu, Mar 06, 2008 at 02:32:24PM +0100, Vincent Palatin wrote:
> > My personal preference is 
> > compilers that not only warn a lot but also treat every warning as fatal
> > error (haven't found this option in gcc).
> 
> GCC option is "-Werror"

Bad idea, IMHO. There are various warnings right now, so this will break
the build for pretty much everybody. Also, the Debian package is
compiled on 12+ architectures for example, and each probably exhibits
different warnings for different targets, endianness etc.

http://buildd.debian.org/pkg.cgi?pkg=openocd

So I would not make -Werror the default (but it's a good idea to remove
as many warnings as possible, of course).
