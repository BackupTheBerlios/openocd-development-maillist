On Sunday 02 March 2008 09:32:41 Øyvind Harboe wrote:
> Could someone clarify the difference and some rational
> arguments w.r.t OpenOCD should use?
>
> My impression is that the latter is supported by GCC and any
> other compiler that OpenOCD could conceivably be compiled
> with, but that it is a little bit less quaint for C++ programmers.
>
> I don't think there are any observable differences in generated
> code.

ANSI C is C89, which is still very much the same as the ISO C90.
C99 brings many enhancements, some of which are broken in GCC. There's a list 
on C99 compliance at the GCC website:
http://gcc.gnu.org/c99status.html

I think any C99 feature that's known to work with gcc 3.3+ might be ok, but 
there's a always certain chance that this isn't stable yet, and might break 
on a subsequent compiler version.

The OpenOCD already makes use of some enhancements, like the designated 
initializers (.poll = arm7_9_poll,).

There are certain features that shouldn't be used:

- C++ style comments
- Variable length arrays (broken anyway)
- Wide chracters
- Basically anything that's exotic enough to cause a) trouble reading the code 
or b) trouble compiling the code
