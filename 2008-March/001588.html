Committed.

Added jtag_khz. It's the same as jtag_speed, except that it takes
KHz as arguments. JTAG interfaces can implement a jtag->khz()
translation function that translates KHz to speed value. By default
jtag_khz will show an error message if jtag->khz() is not implemented.

The motiviation for adding jtag_khz is that it will be used in
the target library to describe the requirements of the target
w.r.t. communication speed. The user can override the
target script statement with jtag_speed/khz approperiate to
his interface.
