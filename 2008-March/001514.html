I don't quite see how this affects target_read/write_buffer(). Perhaps
we should just define that >1 count target->type->read/write_memory()
just needs to service the request as quickly as possible?

Then there would be no error messages involved.

Perhaps target_read/write_buffer() should be the default implementation
of a new fn to target->type?

That way the target can implement any method it wants to service
target_read/write_buffer().

 * possible
