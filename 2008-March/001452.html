Just noticed that the CFLAGS has been changed for the target directory -
personally i am against this.

Any developer will override the CFLAGS anyway, to change optimisation/debug
info etc - as i do for example.
./configure ... CFLAGS="-O0 -g -Wall"

I am favour of removing the default -Wall from this commit, the only flags
that should be added by default are those required to build the project
correctly. Warning etc are optional.
