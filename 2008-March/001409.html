McCleary Mark wrote:
> The read memory/write function for arm11 passes back through the RTI state on retries.  This causes the instruction to reexecute. 
Only the first call to arm11_add_dr_scan_vc()
should end pass through RTI state.  The retries should pathmove() to PD
state avoiding the RTI state.
This simple involves unrolling the first call to arm11_add_dr_scan_vc
from the loop (making the loop
a while{} loop instead of a do{}while() loop.
