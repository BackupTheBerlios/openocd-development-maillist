Hi,

attached is a patch for cortex_m3 adding:
- 16 and 32 bit unaligned accesses supported
- uses packed transfers for 8/16bit read/writes greater than 4bytes
- 8/16bit transfers now use address auto increment

This does raise one issue with openocd, currently openocd handles any
unaligned 16/32bit access with byte read/writes - with cortex this is not
required and is handled by the cortex driver.

The next step is to allow openocd to pass all memory accesses to the cortex
driver - for functions like target_write_buffer/target_read_buffer.
Perhaps we can add a new option variable to the target structure -
ALLOW_UNALIGNED_ACCESS for example.

Michael,
Are you ok with the attached patch?
It has been heavily tested on stm32 but not luminary.
