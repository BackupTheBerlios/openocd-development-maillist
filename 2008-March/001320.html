What is the point of this? I use // comments to signify experimental 
changes in progress vs. /**/ that signify comments. Your patch removes 
that important distinction.

This code is not ANSI C compliant by any means. Changing comments won't 
change that. And all compilers I know accept // while many do not accept 
lots of the GCC specific stuff in the code.



Michael
