Tightening the JTAG API to make it more clear what is supported
and make implementing drivers easier.

Not tested yet.

- jtag_add_reset() now returns void. Callers are required to check
capabilities before invoking jtag_add_reset(). Less awkward
and complicated code + all jtag_add_xxx() functions now have
the same way of handling errors. If jtag_add_reset() is called
with illegal arguments an ERROR("BUG:xxx") is reported.
- TAP_SD/SI are now illegal end states and this is documented and
caught w/test code. Reported as ERROR("BUG:...")
- bitbang.c no longer supports TAP_SD/SI as end states., which
simplifies  the code.
- added comments to bitbang.c.
