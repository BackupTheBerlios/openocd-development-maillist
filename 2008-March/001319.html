The attached patch fixes compile errors when you use the -ansi gcc flag.

Tested with:

CFLAGS="-ansi" ./configure --enable-usbprog --enable-parport
  --enable-parport_ppdev --enable-ft2232_libftdi --enable-amtjtagaccel
  --enable-ep93xx --enable-at91rm9200 --enable-gw16012
  --enable-presto_libftdi

After applying the patch the code builds fine (no errors), but there are
still some warnings left. If you want to see a lots more of those, add
-pedantic to the CFLAGS.

Two simple things needed to be fixed:

 - Use C-style comments instead of C++-style comments.

 - Use '__inline__' instead of 'inline'.
