I think the implementation is pretty much sussed now that the
search path is passed on the command line.

Some further thoughts on how to add a target config script under
Subversion:

- PKGLIBDIR is placed at the end of the search path always
- Add a new PKGLIBDIR/targets dir that contains configuration files that only
  have target config files. The naming of these files are xxx.cfg + e.g.
  xxx_init.cfg for the reset init script. Example of usage of reset init script
  would then be:

  target_script 0 reset targets/h2294_init.script

- PKGLIBDIR/targets contains files that are under subversion. Hopefully
  we'll see lots of such patches for all sorts of targets.
- add a command line argument to execute any number of config scripts
  upon startup  in the order specified on the command line
- fix command.c to allow any command to be executed during the
  config stage and not only config commands. E.g. setting up
  hw/sw breakpoints is part of target configuration, but the hw/sw arm7_9
  config commands are COMMAND_EXEC and thus unavailable from
  a config script today.
