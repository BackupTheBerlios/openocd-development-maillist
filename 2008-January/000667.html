On Jan 17, 2008 11:37 PM, Øyvind Harboe <oyvind.harboe at zylin.com> wrote:
> Some further thoughts on how to add a target config script under
> Subversion:
>
> - PKGLIBDIR is placed at the end of the search path always
> - Add a new PKGLIBDIR/targets dir that contains configuration files that only
>   have target config files. The naming of these files are xxx.cfg + e.g.
>   xxx_init.cfg for the reset init script. Example of usage of reset init script
>   would then be:
>
>   target_script 0 reset targets/h2294_init.script

I don't think that PKGLIBDIR is the correct place for script files.

They should go in either ${sysconfdir}/openocd or ${datadir}/openocd.

BTW: the xscale/debug_handler.bin file should probably go into
${datadir}/openocd instead of PKGLIBDIR since it is not a host lib.

http://www.gnu.org/software/autoconf/manual/standards/Directory-Variables.html#Directory-Variables
