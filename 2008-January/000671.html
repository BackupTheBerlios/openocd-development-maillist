Hi,

This patch adds the "--search|-S" command line option. Each instance
of this option should be a single directory to search for config or
script files. Two directories are always searched: the current working
dir and PKGDATADIR (as defined by autoconf).

This patch also enables passing "--file|-f" multiple time to load more
than one config file. The config files will be loaded in the order
they are given on the command line. If no config file is specified,
the default of openocd.cfg still works.
