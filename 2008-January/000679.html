Some fixes roughly in line with previous discussions:

- allow all commands to be executed the config stage,
e.g. setting up sw/hw breakpoints, etc. the alternative
would have been to hunt down *all* places COMMAND_CONFIG
is used too conservatively when COMMAND_ANY would have
been the right thing to use. That is more work + creates a
messy patch.
- flash auto_erase on/off can now be executed during
config stage
- help is now available for all commands. Previously
COMMAND_CONFIG commands were filtered out. Some
users are quite comfortable w/help & telnet, so having
help available for COMMAND_CONFIG commands
is convenient. This also encourages a small help text
for config commands, which will be welcome.
