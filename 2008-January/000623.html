>

> 
> What happens when you change ft2232.c to the below?

XScale debugging fails, and some (X)SVF files can't be executed anymore (although XSVF could be broken, according to one email report).

I don't have the XScale manuals here right now, but when reading some register (the RX/TX data, iirc), the handshaking flag only gets updated when following a specific path through the statemachine. The default state moves implemented in the OpenOCD weren't sufficient.

XSVF files may specify arbitrary paths through the statemachine. The OpenOCD tries to catch these paths using its default state moves, but if a pathmove can't be expressed as part of a statemove it fails.
