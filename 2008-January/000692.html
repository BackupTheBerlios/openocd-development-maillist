I'd like to change OpenOCD to run a flash probe automatically.

Any objections?

I raised this problem a while back:

https://lists.berlios.de/pipermail/openocd-development/2007-December/000462.html

The attached patch fixes a crash + adds autoprobe when getting the bank.
A cursory inspectiono of get_flash_bank_by_address/num() indicates that this
change is reasonably safe. No recursive stack explotions that I could
spot, all sites check for NULL return value.

Probably drivers should made able to distinguish between an explicit and
implicit probe to improve performance, but the patch should be a good
first step since it fixes the crashes and makes OpenOCD a little bit
easier to use.
