Hi,

This patch allows checking for config/script files in multiple places.
If the OPENOCD_PATH variable is set in the environment, extra
directories will be searched when opening config/script files.

This was a quick hack last night. It works on my Linux box. Don't know
if Windows supports the getenv() function. Don't know if this is the
best implementation. Just passing it on in the hope that it is useful.

This enables me to split up config and script files and place them all
in common directories without having to hard code where the sub parts
are located. I start up openocd with a make rule that sets
OPENOCD_PATH.
