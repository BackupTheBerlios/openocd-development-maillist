On Jan 18, 2008 6:57 PM, Theodore A. Roth <taroth at gmail.com> wrote:
> On Jan 17, 2008 11:43 PM, Dominic Rath <Dominic.Rath at gmx.de> wrote:
> > - There could be all kinds of hidden problems when allowing all commands to
> > execute during configuration (Øyvind mentioned that change in some post). The
> > daemon startup pretty much relies on CONFIG commands executing before EXEC
> > commands.
>
> Could this be mitigated by caching commands in three lists (CONFIG,
> EXEC, ANY) before actually handling them?

No, that's not the problem. The problem is the operation that the
commands perform. Some EXEC command operations might expect the
CONFIG phase to be complete. I don't have a specific example of
an EXEC command that wouldn't produce a sensible error message
and yet do something horrible(i.e. crash), but I'm sure there might
be a case or two.
