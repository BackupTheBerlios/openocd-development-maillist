Here is my plan to remove the error_handler.

1. move verify_ir_capture to the application level. Normally setting
   check_value/mask=in_value/mask will be sufficient when verify_ir_capture
   is 1.

2. #1 makes it possible for arm926ejs.c to use the in_handler to implement
   it's special verify_ir_capture.

3. arm_jtag.c default error handler is to ignore the ir capture error. Delete
   this handler. Users must set verify_ir_capture=0 if they want to ignore
   ir capture errors.

4. arm9tdmi.c error handler currently always fails and it will continue to do.

Sounds good?

This is more mop-up work to simplify the jtag_xxx() layer to
allow for hw + sw implementations and also to make arm926ejs.c
use more exercised code paths + reduce complexity of OpenOCD.
