On Sunday 28 March 2010, Øyvind Harboe wrote:
> BeagleBoard has other special requirements. It has a very special
> JTAG setup... JTAG router? icepick? or whatever it is called. A lot of work
> was done to support it, but I haven't heard anything lately.

No known bugs in the ICEpick support.  "It Just Works" when it's set up...
I used it last week on a DaVinci DM355 ... there are three other OpenOCD
targets now using it (including OMAP3).  Presumably OMAP4 is the same...

However ,f you can get docs out of TI for how to ask ICEpick to reset
chips .... that would help with some of the other chips using ICEpick,
and will help in configurations that don't have SRST (like Beagle or any
other systems using only the TI-14 JTAG connector).

- Dave


p.s. there's also the old/original ICEpick as seen on OMAP1 nd OMAP2, but
not much demand to support that in OpenOCD.  Those modules are incompatible
with the currnt stuff.

- Dave
