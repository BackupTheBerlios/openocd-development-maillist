I've started to write up a branch to retire scan_fields->tap.

jtag_add_xx_scan now takes a single tap as it's first argument.

multiple taps in the same scan was never used in the code, who knows
if that code path worked.

The motivation is to delete code & make the code simpler.

jtag_add_xx_plain_scan can be used for fancy stuff, should it
ever come up.

See last commit of this branch for WIP:

http://repo.or.cz/w/openocd/oharboe.git/shortlog/refs/heads/simplifyscan

Comments?
