On Tuesday 02 March 2010, simon qian wrote:
>  SWDIO and TMS maybe is not the same PIN.
> SWDIO can be multiplexed with TMS, but it's not that it must be multiplexed

The DAP-lite TRM, in section 2.2.5, has the best presentation of how to
switch between JTAG and SWD modes that I've seen.  It talks about one pin:

	SWJ-DP enables either an SWD or JTAG protocol to be used on the debug port.
	To do this, it implements a watcher circuit that detects a specific 16-bit
	selection sequence on the SWDIOTMS pin:

I saw reference to those sequences in one version of the Cortex-M3 TRM,
(which necessarily relied on "one pin" too).
	


> As I remember, there are 3 choices in the ADIv5 document.

This isn't part of the ADIv5 spec...

The place I've seen pinouts discussed is "CoreSight Design Kit
Technical Reference Manual" ... an extract of ARM DDI 0314C3
can be downloaded from e.g. the LM3S811 page at luminary.com.

Section A.2 of that document discusses connector pinouts, and
Table A-1 there says the pin name is "TMS/SWDIO" ...

Of course, at the chip level, chips will integrate one of
three modules:  SW-DP which has SWDIO; JTAG-DP with TMS; or
(as noted above) SWJ-DP which muxes both signals to the same
pin.  (Maybe those are the choices you're thinking of?)  And
in the same way, debug hosts might only support one of those
options ...
