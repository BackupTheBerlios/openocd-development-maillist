Hi everybody,

I've just committed my local changes to the SVN tree. See the SVN log for a 
full list of changes.

To Spen / ARM966E-S users:
I've cleaned up the handling of EmbeddedICE registers. The register size of 
the debug control and status registers is now determined by examining the 
value of the DCC Comms Ctrl version field, which proved to be reliable among 
all the cores I have access to myself (contrary to the documentation, which 
isn't...).
The docs on the ARM966E-S cores isn't clear - revisions 0 and 1 are version 
b0011 (3) according to the documentation, but the ARM9E-S TRMs (R1 and R2) 
don't list that version. Would be nice if you could quickly test with the 
current code, and send me a complete log (just startup and a 'halt' should be 
fine). The new code also removes the hack for disabling the monitor mode, as 
this should happen automatically once the debug control register is first 
written, if the size of that register is correct, i.e. if it includes the 
monitor mode bit.

To Magnus:
I would like to merge the XScale and Cortex-M3 branches soon, as the separate 
branches don't seem to attract a lot of testers. Do you have any patches that 
should go on top of the current Cortex-M3 stuff in SVN?

To All:
If there are no major problems with the new EmbeddedICE register handling 
(that means if the docs are reliable, and the EmbeddedICE version number can 
be relied upon), I would like Michael to create a new binary release for this 
version, so it would be nice if you could test this code on the targets you 
all have access to.
So far, I've verified it on: LPC2294 (ARM7TDMI-S R4), AT91SAM7X (ARM7TDMI R3) 
and CS-EP9301 (ARM920T R1)

Best regards,

Dominic
