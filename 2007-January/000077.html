Hello,

I got the information from Jörn that he has some problem
to use the dump_binary command with a SAM7 cpu.

I have tried it here too, with a STR712 and LPC2148:

dump_binary dump.bin 0x0 0x40000

produce the output file dump.bin without any problems.
It takes about 12 seconds.

If I try this now with an SAM7S256:

dump_binary dump.bin 0x100000 0x40000

OpenOCD is hanging, I could dump some smaller size like 32 or 1204,
but not 0x40000.

Could someone test it with an SAM7 too?

Here I have used the JTAGkey and OpenOCD in revision 124.
