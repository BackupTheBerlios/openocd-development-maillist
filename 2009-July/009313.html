On Tuesday 07 July 2009, Øyvind Harboe wrote:
> > It's very rare that any chip config file needs to say anything
> > about resetting ... since most of the time it's a function of
> > board wiring.
> 
> Except lots of these chips have srst_pulls_trst wired on
> the *inside* of the CPU part....

In which case it's obvious that a *board* level reset_config
policy doesn't accurately say what's going on.

That quirk should probably become a target attribute.  That
way, recovery would be specific to that target ... instead
of getting wrongly applied to targets where the debug logic
stays properly set up after SRST.
