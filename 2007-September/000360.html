On Sep 25, 2007, at 6:20 PM, Jerry Dueitt wrote:

> I've been using OpenOCD very successfully with AT91SAM7S parts
> succesfully on windows based machines for quite some time now. I
> recently made the switch to a Mac on my personal PC at home. I'm
> preparing to start doing some personal work with some ARM parts. I was
> wondering if there are any known issues with compiling OpenOCD and
> using it on a Mac.
> -Jerry.

	I have been able to build it on an Intel Mac running 10.4.9 using  
the GCC 4.0.1 toolchain.  Have it working with a Luminary Micro board  
(DK-LM3S828) using an Amontec JTAGkey and the FTDI USB libs.  Just  
got started with it but so far so good, I can erase&burn flash, start  
and stop, looks good.

	What I haven't yet gotten running natively is the GCC build that  
speaks Cortex-M3/ARM-Thumb-2; until I get that figured out I am  
running the stock CodeSourcery Linux binaries in a VMWare+Fedora7  
setup.  The shared folder support in VMWare makes it pretty easy to  
set up a workspace spanning the two worlds.
