Attached are a couple of patches to image.c
First one adds reading standard address Record (02).

The second change is very strange!!
I noticed that large s19/hex files (>500kB) took absolutely ages (>30sec) to
read under the present scheme. Some old code I had wrote read the same file
in approx 1sec - it used fgets to read.

Changing to using fgets instead of fread cured the speed problem, but
another solution was to memcpy from the file buffer to a local line buffer.
I chose the second method for minimal changes to the codebase.

Thought it might be a win32 thing but I get the same behaviour under
fedora7.

Has anyone come across this before? I certainly haven't.

If everyone is happy I will commit.
