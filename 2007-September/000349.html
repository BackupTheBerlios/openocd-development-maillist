Hi,

I have attached a patch for flash auto erase, my other s19/hex patch is also
included.
Simply it adds functionality to auto erase the flash sectors before a flash
write command.

By default this option is enabled, if the user performs a erase by the
normal flash erase command then the auto erase is switched off. Also auto
erase is disabled for gdbserver flash writing.

New command: flash auto_erase <on|off>

This has been added only for the write_image command, write_binary has not
been changed (manually need erasing) at the moment.

May be time to merge the write_binary command to call the same process as
write_image - if no objections I will do this aswell.

If there are no objections then I will commit patch.
