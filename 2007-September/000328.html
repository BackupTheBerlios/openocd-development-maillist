By default the telnet server only allows one connetion. Is this 
restriction really necessary ?

I dont think many will try to make more than one telnet connection to 
openocd, but there are cases where it could be useful to have a timed 
script doing automatic things over telnet at the same time where a 
standard interactive telnet session handles user commands.  So the 
default limit of allowed telnet connections can be increased.
