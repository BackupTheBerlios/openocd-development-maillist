FWIW... I'm not familiar with i.MX, but bootstrapping Linux (or any
full-featured OS) using the debug hardare can be tricky on some processors. 
Some actions taken by system software may cause problems; for example, if the
kernel puts the processor into sleep mode in the idle loop, or moves or updates
the vector table (which causes a problem on xscale).  I'd read what the data
sheet says about the debug unit, then mindful of that, take a look at what the
kernel does.
