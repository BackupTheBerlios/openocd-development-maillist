> I do write SDRAM setup code, and believe me, I would rather 
> skip re-doing that in the form of a openocd init script if I 
> can. On the IXP42x, getting the SDRAM up is quite easy, but 
> on some DDR1/DDR2 controllers I have seen that require DQS 
> calibration depending on the board layout, it is a lot easier 
> to skip that step and let the bootloader do the work.
> 
> However, to get the bootloader up and running, you need to 
> write to flash - and writing with target code is a lot faster 
> than without, so when the target code can be run from 
> CPU-internal SRAM/cache, that means I can skip DDR setup and 
> save a lot of work.
> If necessary, it will need to be done, but I see no benefit 
> in ditching the current, compact target code in favour of 
> something bigger and slower that requires me to setup DDR 
> memory before use.

I'm 100% with you; it's a pain to have to setup SDRAM/DDR in firmware
*and* JTAG scripts. The downside is you need two scripts; one for a bare
target (without a bootloader) and another for the target with a
bootloader. Even so, that's much less painful.
