I think this might be a better answer - and I think it fits what is 
already in place.

First -
        arm966 = armv5te
and   arm946 = armv5te

Step 1

  change/rename arm966 to "armv5te'
  Hence, "armv5te" could be specified as a target.

Step 2

  create another target structure
  With the *ASCII*NAME* = 'arm966'
   But all other functions identical to armv5te

Step 3
  Likewise for arm946

Step 4
   Any more ... just add another structure...

Notes:
1)   If user specifies target "armv5te"  - the true target (ie: 946 vrs 
966) is the "varient" name

2)   User can always specify 'arm966' as the target - because the ascii 
string in the structure will match.
