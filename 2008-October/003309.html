One concern I have about testing is availability of targets.

There is, for all practical purposes, an infinite number of combinations
of CPU cores & PCBs out there.

Hopefully as OpenOCD matures, the differences will matters less.

Nothing will replace test farms and we have a sizeable collection
of targets that we intend to grow over time. (Thanks to everybody who
has sent us PCBs!)

However, perhaps something can be done in addition to
these test farms?

Some ideas:

1. add a replay capability. Record all communication between
target and all commands issued. The idea is to allow openocd
developers to replay a test sequence that users have performed.

2. add a TCP/IP based JTAG interface + add TCP/IP port to
OpenOCD to be accessed using said TCP/IP JTAG interface.
The idea is that users make their PCBs available via e.g.
reverse port forwarding & SSH or simply opening up a
port in a firewall.

Other ideas?

I think perhaps #2 is the most powerful, but still quite tedious.
