Hello,

My .gdbinit script:

target remote 127.0.0.1:3333
monitor reset halt
load test.elf

Steps to reproduce:
OpenOCD is started (openocd-init.txt).
start gdb the first time (openocd-first.txt) (gdb script is executed
as expected)
quit gdb
start gdb the second time (openocd-second.txt)
Now openocd crashes:
Error:   729 193470 register.c:102 register_get_arch_type(): BUG:
encountered unregistered arch type 0x00000000
It's caused by GDB sending "P19=00000001".

If I don't reset the target before using the load command or loading
the efl file with GDB's file command first OpenOCD doesn't crash.
Any idea why a reset causes GDB to send the P19 command?

When I try adding "set remote P-packet off" to the init scrip GDB
sends a very long G command (see openocd-segfault.txt), resulting in a
Segmentation fault.

Software:
OpenOCD Version is SVN 1021 (But it's not a new Bug). with GDB IO Debug enabled
GNU gdb (Sourcery G++ Lite 2008q1-126) 6.7.50.20080107-cvs

Hardware: LMI LM3S8962 EVK (Cortex M3)

greetings
Freddy
