are you sure this works for both arm7/9 and cortex cores - i am not.
as setting org.gnu.gdb.arm.core-v7m is telling gdb that we have a v7 core
even if a 7tdmi is attached.
all the registers will be incorrect.

The only safe way of doing this is leaving it upto the user to specify, as
gdb does.
also this will break every version of gdb that does not understand
org.gnu.gdb.arm.core-v7m, eg. mainline.

It would be good if gdb gave us a response to this message, but it does not.
we need something like gdb_target arm_v7.xml that contains the register
descriptions to send to gdb - i have looked into this just never found the
time to finish.

The only issue with sending a xml is that openocd registers will need to be
dynamic as the user can add their own registers as part of the target xml.
