Seems reasonable - debugging/developing bootloader code without a JTAG debugger is a real PITA (BTDT).

> The platform I am looking for is IXP435/432. I am wanting to use the
> MAJIC JTAG from MentorGraphics/ EPI. see the link below. My company
> brought this JTAG 3 years back and missing with the licence information.
> Will I be able to use it with openOCD in thsi case?

I am afraid not. The Majic is a complete JTAG-to-Ethernet debug adapter that does most of what openocd does inside the Majic box, so you just need the debugger (you might check which debuggers are supported - maybe gdb can directly talk to the box?). If gdb does not work, then the box can probably only be used by getting a license for a supported debugger.

OpenOCD's purpose is to replace boxes like the Majic: it only requires some simple JTAG adapter (either a simple parallel-port wiggler, FTDI2232-bases USB, or similar solutions - the supported adapters should be listed in the manual).
