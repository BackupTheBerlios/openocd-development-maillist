If you are going to solicit the help of the mailing list, you'll need to
explain what sort of CPU this is.

Since OpenOCD is GPL, the idea is to share e.g. target support.

You can either start by having a look at e.g. the mips target and
then secondarily the arm7 target. The MIPS target will show you
how to implement a completely new architecture and the arm7
is the most tested target. There are no written guides for this
that I know of. Of course this is an open source project and we
would gladly commit any decent documentation to openocd.texi
on this.
