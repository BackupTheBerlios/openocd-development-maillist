Hi,
I tested with IAR EWARM 5.11KS thru GDB on LPC2138.
If I debug in Flash, everything is OK.
But if I debug in RAM, svn later than 919 will fail.

With svn919, it's OK but EWARM will use single-stepping due to lack of breakpoints.
With svn 920, it's ONLY OK for the first time, openocd need to be restarted to function well.
With svn 986, it's ONLY OK for the first time, power of target need to be toggled to function well, of course openocd should be restarted.
If it's OK with svn later than 920, EWARM will use breakpoint instead of single-stepping.

When error occurs, error message is same:
Error:  unknown character 0x24 in reply, dropping connection
Warning:ignoring character 0x6d
Warning:ignoring character 0x31
Warning:ignoring character 0x2c
Warning:ignoring character 0x31
Warning:ignoring character 0x23
Warning:ignoring character 0x66
Warning:ignoring character 0x62
