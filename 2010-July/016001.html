On Sun, Jul 4, 2010 at 10:01 PM, Freddie Chopin <freddie_chopin at op.pl> wrote:
>> Your 64bit packages need to use the 64bit of libftdi dll so I can not
>> test with libftdi-0.18 yet.
>
> That's true - I have compiled both libftdi and libusb-win32 for the x64
> packages with x86_64-w64-mingw32 toolchain.
>
> Is it possible to install the drivers bundled with OpenOCD
> (libusb-win32_ft2232_driver-100223.zip) in Win7 x64?
>

Yes but you have to either disable the signing check (with F8) or
use test signing (test sign the driver package and then enable
test signing). For Win7 x64, the right hand corner of the screen
will have a small test signing watermark.which is not that obvious.

If you build the svn version of libusb-win32, you can enable
test signing with an option in the build script (make.cmd).

In order to avoid test signing, you have to build against
libftdi-1.0 and then use WinUSB.sys as the driver right now.
You can use Zadig for winusb driver installation.

Or you can wait a bit and wait for the signed release of
libusb0.sys. BTW, we change the name of the 64bit
DLL and driver to be the same as the 32bit (but reside
in different directory) for the v1.x release of libusb-win32.

The new INF wizard or Zadig will create the correct inf
file for WIn32 and Win64.
