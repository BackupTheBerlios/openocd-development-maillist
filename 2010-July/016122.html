You may still want to change to this scheme.

The other way is that you do not need to distribute the
inf file, but rather point the users to either the new
inf-wizard in libusb-win32 (not yet released, but in
svn, soon to be released) or zadig/libwdi. Both
can be used to install libusb-win32 device driver
in a GUI based and rather user-friendly way.

zadig/libwdi: http://www.libusb.org/wiki/windows_backend
new inf-wizard which is based on libwdi as well.
http://libusb-win32.svn.sourceforge.net/viewvc/libusb-win32/branches/libusb-testing/

zadig makes the driver installation for winusb and libusb-win32
much easier and you can also easily switching between in
case you want to switch between libftdi-1.0 and libftdi.
