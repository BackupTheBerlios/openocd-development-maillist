On Thursday 07 May 2009, Martin Panter wrote:
> I never used git bisect so I don't know exactly how it works. 

"git bisect --help" summarizes:

	git bisect start
	git bisect bad		... assuming current is broken
	git bisect good revid	... some known-good version

Then cycles of

	build
	test
	git bisect good		... if it passed, or
	git bisect bad		... else

Until it tells you which one broke things.  Cleanup:

	git bisect reset

Same interface as "svn-bisect" ... except that it works.


It's fairly mindless to use, which if you have time and the
test is simple enough.
