On Saturday 09 January 2010, David Brownell wrote:
> Are there more issues that seem like they need attention before
> we freeze 0.4.0 ?

For the record, the most significant items now left on my own
list of stuff to do before freezing 0.4:

 - That NOR "flash erase_address pad addr length" feature.
   Patch written but needs testing.

 - Test arm920t cp15 access changes ... there was quite a lot
   of change there, possibly enough to cause regression on
   the only platform in the tree (csb337) using that code.

 - If I can verify it works properly on ARM9, merge the last
   bit of support for the reset-assert event:  ARM7/ARM9 glue.
   That works fine for newer ARMs (Cortex-A8, ARM11), but most
   cores are older.  (Completing mostly-merged work.)

 - Sanity testing with more boards, to make sure regressions
   didn't sneak in.  Includes some NAND stuff.

 - Semihosting:  try it out, provide basic docs.  This is a
   pretty significant new feature, and I'd *REALLY* like to
   see some success reports ...

 - There are a few other bits of doc that seem to be missing,
   based on queries from the list.  Notably, single stepping
   Cortex-M3 in the face of IRQs ... a typical task.

I'm sure there are a few issues other folk have been carrying
around.  Please share!

- Dave
