On Saturday 16 January 2010, Paul Fertser wrote:
> > If use .cfg for a AVR8 chip, there will be many such configure
> > files.

> I do not see a problem. Moreover if openOCD implements proper avr

Simon, do the AVR8 chips have proper JTAG IDs, from which that
configuration information can be derived?  Like a table mapping.

Sometimes vendors are a bit weak with making each chip have a
unique ID ... instead several chips have the same ID.  But then
sometimes they compensate by having secondary IDs.

Yes, having a per-chip config file is an *annoying* and somewhat
error prone, but workable, solution.

If OpenOCD were to support some of the AVR8 chips for JTAG debug
(instead of just programming), it would clearly be better to have
some safe way to at least verify such config files.  :)
