When we froze RC1 on 21-December, I said I expected to see at least three
weeks of testing.  Three weeks works out to 11-January, which in some time
zones is tomorrow ...

So I thought a status report and query would be appropriate... please
reply on this thread if you have feedback or more information.

Right now there are some reported-but-unresolved isssues:

 - Crashes in lpc3180 NAND driver, reported by Richard Vegh.
   Looks like small page support for this newish driver has
   never worked.  This should be easy to fix, and I'm inclined
   to require the fix before the final 0.4.0 code.

 - FreeBSD buildfix.  The trivial one seems to be just getting
   the $ocd_HOSTOS variable initialized correctly; #ifdeffable.

 - Misbehavior for server shutdown command.  Øyvind has a patch
   but he's not merged it.

 - MIPS 4K issue with (non?) CFI flash detect on one board;
   and, presumably unrelated, byte order.  I'm not clear on
   status here, but this looked to be a regression.

 - Various NOR structural issues:  (a) wrongly writing ones,
   (b) Øyvind's desire for an "erase_address" option to pad
   to sector boundaries.

 - Various ARMs should be enabling DCC downloads more often,
   for speed.  (Not necessarily release-blockers...)

Are there more issues that seem like they need attention before
we freeze 0.4.0 ?

- Dave
